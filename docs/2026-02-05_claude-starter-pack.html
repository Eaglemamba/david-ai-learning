💎 Part IV：被低估的技巧 Underrated Tips Nobody Talks About

這個章節揭示了六個「隱藏功能」，其中最有價值的是 Token Usage 模型選擇矩陣和 Style Selection 功能。Token 使用策略直接影響成本和效率，而 Style 功能則能改變 Claude 的回應結構。

"Model selection matrix to save on token usage: Haiku 4.5 - switch to this for most of your tasks (80%+ of tasks), Sonnet 4.5 - for 'medium' level tasks only (10% of tasks), Opus 4.5 - ONLY use Opus 4.5 when absolutely necessary (final 10% of tasks)."

📚 核心概念：80/10/10 模型分配法則

這個分配建議反映了成本效益優化原則：

  • Haiku 4.5 (80%)：日常任務、快速問答、簡單生成
  • Sonnet 4.5 (10%)：中等複雜度、需要較好推理能力
  • Opus 4.5 (10%)：複雜分析、關鍵決策支援、創意任務

這與你的「Copilot vs Agent」框架一致：日常 Copilot 用 Haiku，Agent 任務用 Sonnet/Opus。

🛠️ 實務應用：Style Selection

Style Selection 讓你可以改變 Claude 的回應結構：

  • Concise：精簡回答，適合快速查詢
  • Learning：教學模式，適合學習新概念
  • Custom：自定義風格，可以設定特定格式

對於你的 Master Prompt 系統，可以考慮建立對應的 Style 預設。

⚠️ Claude Cowork 的價值

作者認為 Cowork「massively underrated」，是「市場上對一般人最好的 AI 工具」。關鍵建議：

  • 建立專用資料夾給 Cowork 存取
  • 不要給予所有資料夾的權限
  • 這符合「最小權限原則」的安全實踐

🚫 Part V：常見陷阱與解決方案 Common Pitfalls (and how to fix)

最後這個章節列出了四個最常見的使用錯誤。其中 Context Dumping（一次性傾倒大量上下文）和 No Rules（沒有設定限制）是最普遍的問題。作者建議使用 CLAUDE.md 來定義嚴格規則。

"When you dump context into Claude, it can forget earlier/previous context. Always 'drip feed' context step by step. This will decrease your hallucinations and increase output quality."

💡 比喻說明：Context Dumping

想像你在指導一位新員工：

  • 錯誤方式：一次性給他 500 頁的 SOP 說「讀完這些就知道怎麼做了」
  • 正確方式：先說「今天我們學第一章」，確認理解後再進入下一章

「Drip feed」就是這個概念——分批餵入上下文，讓模型能夠正確處理每個部分。

💻 CLAUDE.md 的作用

CLAUDE.md 是一個設定檔，用於定義：

  • Coding style 規範
  • Project constraints 專案限制
  • Output boundaries 輸出邊界

這類似於你的 Master Prompt 系統——預先定義規則可以避免 Claude「漫遊」到不相關的任務，節省 Token 使用。

🛠️ 四大陷阱速查
  1. Wrong Prompting → 參考官方 Prompt Engineering 指南
  2. Context Dumping → 分步驟餵入上下文
  3. Repetitive Prompting → 建立 Skills 或 Notion 資料庫
  4. No Rules → 使用 CLAUDE.md 設定邊界

✏️ 練習題 Practice Questions

Q1 概念理解 解釋「Context Dumping」為什麼會導致 Claude 的輸出品質下降，並描述「drip feed」方法如何改善這個問題。

答案要點：

  • Context Dumping 的問題：當一次性傾倒大量上下文時，Claude 的注意力機制需要在所有資訊中分配權重。早期提供的資訊可能被後來的資訊「稀釋」，導致遺忘或錯誤關聯。這類似於人類閱讀——如果一次給你 100 頁文件，你很難記住第 5 頁的細節。
  • Drip Feed 的改善：分步驟餵入上下文讓每個「批次」的資訊都能被充分處理。你可以確認 Claude 理解了第一部分後再提供第二部分，形成漸進式的知識建構。
  • 實務建議：將長文件拆分成邏輯區塊，每次提供一個區塊並要求 Claude 摘要或確認理解，再進入下一個區塊。
Q2 產業應用 在 CDMO 的無菌充填環境中，如何應用「80/10/10 模型分配法則」來優化 AI 工具的使用？請舉出具體的任務分類範例。

CDMO 應用範例：

  • Haiku (80%)：日常郵件撰寫、簡單資料查詢、會議記錄摘要、SOP 格式檢查、基礎翻譯任務
  • Sonnet (10%)：Deviation 初步分類、Batch Record 異常識別、供應商評估報告初稿、訓練教材生成
  • Opus (10%)：複雜 Root Cause Analysis、製程可行性評估報告、跨部門整合策略文件、客戶 Tech Transfer 文件審閱

關鍵原則：GMP 環境的關鍵決策（如 Batch Release）仍需人類 Copilot 模式審核，即使使用 Opus 也不能完全依賴 AI 輸出。

Q3 批判思考 這篇文章的「資源彙整」形式有什麼優點和局限性？對於進階使用者，還需要補充什麼類型的內容？

優點：

  • 節省搜尋時間，提供結構化的學習起點
  • 涵蓋從初學到進階的資源，適合不同階段的學習者
  • 包含「常見陷阱」，幫助避免新手錯誤

局限性：

  • 缺乏深度分析——列出資源但沒有比較評價
  • 沒有「學習路徑建議」——讀者不知道先讀什麼
  • 部分連結（如 Skills Marketplace）未經品質篩選

進階使用者需要：

  • API 整合的技術深度文件
  • 企業級部署的安全和合規考量
  • 產業特定的應用案例研究
  • Token 經濟學和成本優化的量化分析
