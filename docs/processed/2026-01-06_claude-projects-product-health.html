Part 1 of 4 Section 1: Claude Projects Overview
What is Claude Projects?

Projects lets you create a self-contained space for you or your team to come back to over time. A Project has its own dedicated chat history, a knowledge base of uploaded documents and custom instructions that Claude will consider every time it generates a response.

You can also combine Projects with newer abilities like Skills and Claude for Excel to create artifacts from the output of your Projects that you can share with stakeholders and colleagues.

A project has two key characteristics when you first create it: a name and a description. Once you've created a Project, you'll see the Projects home screen which includes the main chat interface and chat history along with your custom instructions and files / data sources.

📚 Core Concept: Projects 三大支柱

Knowledge Base（知識庫）：上傳相關文件，Claude 會在每次對話中參考這些資料。

Chat History（對話歷史）：所有對話都保存在 Project 內，方便追溯。

Custom Instructions（自定義指令）：設定 Claude 的角色、回應格式和行為準則。

💡 Analogy: 把 Project 想像成一個專案辦公室

Knowledge Base 就像辦公室的檔案櫃，Custom Instructions 是貼在牆上的工作守則，Chat History 則是會議紀錄本。每次你走進這個辦公室（開啟 Project），Claude 都會記得這些背景脈絡。

🛠 CDMO Application

在製藥 CDMO 情境中，你可以為每個客戶專案建立獨立的 Claude Project：上傳 Batch Records、Deviation Reports、製程參數；設定符合 GMP 語境的 Custom Instructions。這比每次重新說明背景省下大量時間。

Part 2 of 4 Section 2: Five Health Indicators
The Five Indicators of Product Health

One of the biggest challenges that product people face when crafting a product strategy is figuring out where exactly to start. A health check gives you a snapshot of where things stand across multiple dimensions today.

This is especially valuable when joining a new company or taking over a product, but regular health checks (quarterly or biannually) keep you grounded even when you've been working on the same product for years.

1. User Data Fundamentals: Total visitors → Active users → Paying users (the conversion funnel), Geolocation breakdown, Device distribution.

2. Value Proposition Clarity: Do users understand what we're offering? Do they actually want it? Measure through PMF indicators: Very Disappointed score, NPS, LTV, CAC.

3. UX Health: Task completion rates, Conversion rates on critical journeys (signup, login, payment), Friction indicators: form validation errors, rage clicks, server-side errors.

4. Business/Commercial Health: Is our monetization strategy clear? Are revenues growing or shrinking? What do sales teams say about market opportunity?

5. Strategic Health: Clear vision, decisions that align with it, ability to say no. Assess differentiation across: Features, Policies, UX, Pricing, Performance.

📚 Core Concept: 健康檢查 vs. 單點指標

傳統 KPI 追蹤往往聚焦單一指標（如 DAU、Revenue），但這容易產生「隧道視野」。5 維度健康檢查的價值在於跨維度連結：UX friction 可能解釋 revenue 下滑，value prop confusion 可能反映在 sales 困難。

⚠ Key Point: Traffic Light Scoring

作者建議使用簡單的三色標記：

  • ● Healthy - 表現良好，維持現狀
  • ● Attention - 需要關注，但不緊急
  • ● Critical - 正在傷害產品，優先處理
🛠 CDMO Application: 5 維度對應

User Data → 客戶 Pipeline、詢價轉換率

Value Prop → 我們的技術差異化（如 SA25 Vanrx、QS-21）是否被客戶理解？

UX Health → Tech Transfer 流程的摩擦點、客戶 onboarding 體驗

Business → 產能利用率、Margin、客戶留存

Strategic → 專注 Fill/Finish 還是擴展上游？與競爭者的定位差異？

Part 3 of 4 Section 3: Implementation Approaches
Approach 1: Single Project

First, create your Project and give it a name and description. We'll need to give the Project custom instructions that tell Claude what role they should take and how exactly they should respond.

Since we're only using a single Project, we need to tell Claude how to categorize each file. Adopt a naming convention that prefixes every file with the correct category. For example: userdata_analytics.xlsx, ux_designfiles.pdf, strategy_competitive.docx.

Approach 2: Multiple Sub-Projects

Set up 5 different sub-Projects for each focus area, then consolidate findings into 1 master Project. This may be better when you want to delegate management to different teams. Design teams manage UX health, commercial teams manage Business Project, and so on.

The Master Project synthesizes summaries from all 5 sub-projects, using prompts like: "Synthesize into a single executive health report with: Overall Health Snapshot, Cross-Dimensional Patterns, Top 3 Priorities, Top 3 Strengths, Leadership Talking Points."

⚖ Comparison: Single vs. Multiple Projects
FactorSingle ProjectMultiple Projects
Setup EffortLowHigh
MaintenanceEasyModerate
Context WindowLarger (all files)Smaller per project
Team DelegationLimitedExcellent
Cross-dimension AnalysisBuilt-inRequires Master Project
💻 File Naming Convention

使用 prefix 讓 Claude 自動分類：

userdata_google_analytics_q4.xlsx
valueprop_customer_survey_results.pdf
ux_heatmap_checkout_flow.png
business_revenue_forecast_2026.xlsx
strategy_competitor_matrix.docx

Pro tip: 可用 Claude Code 批次重新命名現有檔案。

💡 Analogy: 單店 vs. 連鎖店

Single Project 像是一家全能的獨立店面：所有商品都在同一個空間，管理簡單但可能雜亂。

Multiple Projects 像是連鎖店系統：每個分店專精一個品類，總部（Master Project）負責彙整各店報表。規模大時效率更高，但需要更多協調成本。

Part 4 of 4 Section 4: Prompts and Skills Integration
Sample Prompts for Health Checks

Weekly Health Check:

Run a quick health check across all five dimensions based on the latest data. Give me traffic light status for each and flag anything that needs immediate attention.

Cross-Dimensional Analysis:

I'm seeing declining conversion rates in our signup funnel. Analyze this across all five health dimensions—what might be contributing from each angle?

Differentiation Assessment:

Assess our differentiation across Features, Policies, UX, Pricing, Performance. Where are we actually differentiated? Where are we vulnerable?

Monthly Executive Report:

Create a monthly Product Health Report for leadership. Format for a 5-minute read: overall status, top concerns, key wins, and recommended focus areas.

🛠 Skills Integration

Claude Skills 可以將健康檢查結果轉換為可分享的成果物：

  • Excel Skill：產出格式化的數據表格、公式正確的 dashboard
  • PowerPoint Skill：依品牌規範產出簡報，直接用於 leadership meeting

這讓 Product Health Project 從「對話工具」升級為「報告產生器」。

⚠ Key Point: Custom Instructions 結構

好的 Custom Instructions 應包含：

  • Role Definition：Claude 扮演什麼角色（如 Product Health Analyst）
  • Context：產品名稱、Active User 定義、關鍵競爭者
  • Framework：5 維度定義與判斷標準
  • Output Formats：Quick Check、Deep Dive、Executive Summary 各自的格式
📚 Core Concept: Institutional Memory

作者提到一個關鍵價值：「measuring product health is an ongoing practice that compounds over time」。透過 Claude Projects，你建立的不只是一個工具，而是機構記憶（Institutional Memory）。

這種記憶可以存活過團隊更迭、策略轉向、日常混亂。當新 PM 加入時，他們可以立即理解產品的歷史脈絡。

Key Takeaways

Projects = Persistent Context

Claude Projects 整合知識庫、對話歷史、自定義指令，提供持續性的脈絡記憶，是進行長期追蹤任務的理想工具。

5 Dimensions, Cross-Analysis

產品健康不是單一 KPI，而是 5 個維度的交叉分析。維度之間的連結往往揭示根因。

Architecture Fit to Team

Single Project 適合小團隊快速上手；Multiple Projects 適合跨部門協作與權責劃分。

Skills Amplify Output

結合 Excel/PowerPoint Skills，將健康檢查從對話升級為可分享的專業報告。

Practice Questions
Question 1: Concept Understanding
為什麼作者認為 Claude Projects 比起單次對話更適合用於產品健康監測？請說明 Projects 的哪些特性支持這個用途。

Answer:

Claude Projects 適合產品健康監測的三個關鍵特性：

1. Knowledge Base（知識庫）：可以上傳所有相關文件（GA 報表、用戶調查、財務數據），Claude 在每次對話中都能參考這些資料，不需重複說明。

2. Chat History（對話歷史）：所有過往的健康檢查對話都保存在 Project 內，可以追溯歷史趨勢、比較不同時期的狀況。

3. Custom Instructions（自定義指令）：設定好 5 維度框架、Traffic Light 標準、輸出格式後，每次對話都會遵循這些規則，確保一致性。

這三者結合，讓 Claude Projects 成為一個「有記憶的產品顧問」，而非每次從零開始的對話。

Question 2: Practical Application (CDMO Context)
假設你要為你的 CDMO 建立一個「客戶專案健康監測」的 Claude Project，請設計一個適合的 Custom Instructions 大綱，包含角色定義、監測維度、輸出格式。

Suggested Custom Instructions Structure:

## ROLE
You are a Client Project Health Analyst for [CDMO Name], 
specializing in aseptic fill/finish operations.

## FIVE HEALTH DIMENSIONS

1. **Project Timeline Health**
   - Milestone completion rates
   - Schedule adherence vs. original plan
   - Critical path risk indicators

2. **Technical Performance**
   - Batch success rates
   - Deviation frequency and severity
   - Process parameter stability (e.g., SA25 Vanrx)

3. **Regulatory Compliance**
   - Documentation completeness
   - Audit findings status
   - Change control backlog

4. **Commercial Health**
   - Budget vs. actual spend
   - Scope change frequency
   - Client payment status

5. **Relationship Health**
   - Communication frequency
   - Issue escalation patterns
   - Client satisfaction signals

## OUTPUT FORMATS
- **Quick Status**: Traffic light per dimension + top risk
- **Monthly Report**: Dashboard with trends + recommendations
- **Escalation Brief**: Executive summary for leadership
Question 3: Critical Thinking
作者提到 Single Project 架構會「bloat the context window」，而 Multiple Projects 可以避免這個問題。請分析：在什麼情況下 context window 大小會成為實際問題？這個問題的嚴重程度是否被高估了？

Analysis:

Context Window 問題會發生在：

  • 上傳大量 PDF/Excel 文件（如完整年度財報、所有客戶合約）
  • 文件內容高度重複或冗餘
  • 需要 Claude 同時參考多個大型文件進行分析

問題嚴重程度評估：

以 Claude 目前的 context window（200K tokens for Opus），實際上相當寬裕。作者的擔憂可能略為誇大，除非：

  • 你的文件總量超過 150K tokens（約 30-50 頁 dense PDF）
  • 你需要 Claude 精確引用文件中的特定段落

建議：先從 Single Project 開始，只有在遇到「Claude 忘記文件內容」或「回應變慢」時，才考慮拆分為 Multiple Projects。過早優化（premature optimization）是常見陷阱。
