Content

Part I — Dispatch：手機遙控你的桌面 AI

Bilingual Content
Dispatch 讓你與 Claude 維持一個跨裝置的持續對話，手機和電腦之間同步。你從手機傳送任務，Claude 在桌面端使用你的本地檔案、應用程式和工具來執行。
Dispatch gives you one persistent conversation with Claude that syncs between your phone and your computer. You text Claude a task from your phone. Claude runs it on your desktop using your local files, apps, and tools.
把它想像成傳訊息給一個坐在你電腦前的助理。「打開下載資料夾裡的提案，給我重點摘要。」「搜尋客戶來信並整理摘要。」你從任何地方發送訊息，Claude 在你的機器上完成工作。
Think of it like texting an assistant who sits at your computer. "Pull up that proposal in my downloads and give me the key points." "Search my email for anything from the client and make a summary."
📚 核心概念
Dispatch 的本質是手機 ↔ 桌面的持久通道。它不是新的 AI 能力，而是一個「觸發器」——真正做事的是 Cowork。

前提條件：Claude Desktop 必須開啟、電腦必須醒著、需要 Pro 或 Max 訂閱。
💡 CDMO 類比
Dispatch 像是 CDMO 裡的遠端批次指令系統——你可以從辦公室（手機端）對產線（桌面端）下達指令，但所有實際操作仍然發生在產線現場，所有 GMP 記錄也留在產線端。Remote trigger, local execution。

Part II — Cowork：讓 Claude 從聊天機器人變成執行代理

Bilingual Content
Cowork 是讓 Dispatch 成為可能的引擎。它是 Claude Desktop 裡的一個模式，把 Claude 從聊天機器人變成可以在你電腦上實際操作的代理人。
Cowork is the engine that makes Dispatch possible. It's the mode inside Claude Desktop that turns Claude from a chatbot into an agent that can actually do things on your computer.
在 Cowork 模式下，Claude 可以：直接讀寫你電腦上的檔案、建立帶公式的 Excel 試算表、從零開始製作 PowerPoint 簡報、使用子代理人同時執行多個任務、連接 Google Drive / Gmail / Slack 等工具、處理長時間運行的任務而不會超時。
In Cowork mode, Claude: Reads and writes files on your computer directly, creates real Excel spreadsheets with formulas, builds PowerPoint presentations from scratch, runs multiple tasks at the same time using sub-agents, connects to your Google Drive, Gmail, Slack, and other tools, handles long-running tasks without timing out.
Dispatch 是遙控器，Cowork 是引擎。
Dispatch is the remote control. Cowork is the engine.
⚠️ 重點提示
Chat Mode vs Cowork Mode 的關鍵差異：
Chat = 問答介面（只回答問題）
Cowork = Agent 模式（實際操作檔案、連接外部工具、執行多步驟任務）

這個區分很重要：Cowork 本質上就是 Anthropic 對「Agent Mode」的產品化包裝。
🛠️ 實務應用
Sub-agents（子代理人）是 Cowork 的隱藏殺手級功能。Claude 可以同時啟動多個子任務並行處理——這在處理「搜尋 + 整理 + 產出報告」這類複合任務時特別有用。

對 CDMO 場景：想像同時查詢 TrackWise 偏差、搜尋 SOP 歷史版本、整理趨勢圖——如果這些資料源都有 connectors 的話。

Part III — Plugins：預打包的工具套件

Bilingual Content
Plugins 是賦予 Claude 在 Cowork 中新能力的工具套件。每個 Plugin 包含三個元素：Skills（特定任務的專門指令）、Connectors（連接 Slack / Gmail / Google Drive 等外部應用）、Sub-agents（處理部分工作的小型 AI 代理）。
Plugins are bundles of tools that give Claude new abilities inside Cowork. Each plugin includes: Skills (specialized instructions for specific tasks), Connectors (links to external apps like Slack, Gmail, Google Drive), Sub-agents (smaller AI agents that handle pieces of work).
涵蓋銷售、行銷、財務、法務、工程、HR、設計、營運和資料分析等領域的預建 Plugins，一鍵安裝。你也可以自建 Plugins 或上傳團隊成員的自訂 Plugins。
There are plugins for sales, marketing, finance, legal, engineering, HR, design, operations, and data analysis. You install them with one click. You can also build your own plugins or upload custom ones from teammates.
⚖️ Plugin 結構拆解
Plugin = Skills + Connectors + Sub-agents

可以類比為 CDMO 的「設備資格確認套件」——不是單一文件，而是包含 IQ/OQ/PQ 範本（Skills）、設備連接協議（Connectors）、和各部門負責人（Sub-agents）的完整套件。

安裝路徑：Cowork → Customize → Browse Plugins → Install
⚠️ 重點提示
Plugin 的「一鍵安裝」設計降低了門檻，但也帶來風險：你不一定知道 Plugin 內部做了什麼。在合規環境中，這等同於安裝未經驗證的軟體——需要先了解 Plugin 的權限範圍和資料存取行為。

Part IV — Skills：教 Claude 按你的方式做事

Bilingual Content
Skills 教 Claude 如何正確地完成特定任務。它們不只是文字檔——而是可以包含腳本、範本、參考資料和範例的資料夾。
Skills teach Claude how to do specific tasks the right way. They're not just text files - they're folders that can include scripts, templates, reference data, and examples.
在 Cowork 中輸入「/」就能看到所有可用的 Skills。Claude 也會自動偵測何時需要載入相關的 Skill。內建 Skills 包括 Excel 建立與操作、Word 格式化、PowerPoint 簡報建立、PDF 處理。
When you type "/" in Cowork, you see every skill available to you. Claude also detects when a skill is relevant and loads it automatically. Built-in skills include: Excel spreadsheet creation and manipulation, Word document formatting, PowerPoint presentation building, PDF creation and processing.
但真正的威力在自訂 Skills：你可以教 Claude 你的品牌語調、報告格式、內容流程、客戶工作流程。以 ZIP 檔上傳後，每個對話都能使用。
But the real power is custom skills. You can teach Claude exactly how your business works - your brand voice, your reporting format, your content process, your client workflow. Upload them as a ZIP file and they're available in every session.
💡 Skills 的層級關係
Skills 是 Plugins 的子元素——一個 Plugin 可以包含多個 Skills。但 Skills 也可以獨立存在（不一定要透過 Plugin 安裝）。

類比：Skills = SOP（Standard Operating Procedure）
Plugin = SOP + 設備連接 + 人員分工 的完整套件

你已經在用的 artifact-skill、amaran-presentation 就是 Custom Skills 的概念——只是目前在 Claude Projects 裡，未來可能遷移到 Cowork Skills 系統。
🛠️ 啟用路徑
Settings → Capabilities → 開啟 "Code execution and file creation"
→ Customize → Skills → 勾選需要的 Skills

自訂 Skills 上傳：ZIP 檔格式，包含 SKILL.md + 範本/腳本/參考資料

Part V — Scheduled Tasks：自動排程

Bilingual Content
Scheduled Tasks 讓你設定自動重複執行的工作。描述一次任務、設定排程，Claude 會按時處理。
Scheduled tasks let you set up work that runs automatically on a recurring basis. You describe the task once, set the timing, and Claude handles it on repeat.
排程任務可以使用 Cowork 的所有功能——你的檔案、Connectors、Plugins 和 Skills。只要你的電腦是醒著的且 Claude Desktop 正在運行，任務就會按時執行。
Scheduled tasks use everything Cowork has access to - your files, connectors, plugins, and skills. They run as long as your computer is awake and the Claude Desktop app is open.
⚠️ 重要限制
「電腦必須醒著」是架構瓶頸。Scheduled Tasks 依賴本地機器運行，不是雲端執行。這意味著：
- 筆電合蓋 = 任務停止
- 網路中斷 = 任務失敗
- 無法保證 SLA（服務等級協議）

這與 Make.com / n8n 等雲端自動化工具的架構根本不同。適合「nice-to-have」的自動化，不適合「mission-critical」的排程。
🛠️ 排程設定
在 Cowork 中輸入 /schedule
或點擊側邊欄 "Scheduled" → "+ New task"
可選頻率：每小時、每日、每週、工作日、手動觸發

Part VI — 系統如何串接：五層協同工作

Bilingual Content
大多數人忽略的重點：這些不是獨立功能，而是同一系統的不同層次。
Here's the part most people miss. These aren't separate features. They're layers of one system.
當你從手機 Dispatch 發送「整理我的每週銷售報告」時，完整流程是：Dispatch 把訊息送到桌面 → Cowork 接收並啟動 session → 相關 Plugin 連接 Google Sheets 和 CRM → 相關 Skill 按你的偏好格式化報告 → Cowork 產出完成的文件 → 你在手機上看到結果。
When you send a Dispatch message from your phone saying "compile my weekly sales report": Dispatch sends the message to your desktop → Cowork receives it and starts a session → The relevant plugin connects to your Google Sheets and CRM → The relevant skill formats the report the way you like it → Cowork produces the finished document on your computer → You see the result on your phone.
一則訊息，五個系統協同運作。這就是整個生態系統。
One message. Five systems working together. That's the ecosystem.
📚 架構分析
Spisak 用「五層」來描述，但實際上更準確的分類是：

Interface Layer（介面層）：Dispatch（手機）、Cowork（桌面）
Capability Layer（能力層）：Plugins、Skills
Orchestration Layer（編排層）：Scheduled Tasks

這個架構與 Anthropic 工程部落格的 "Building Effective Agents" 思路一致——把 Agent 分解為可組合的模組，而非單一 monolithic 系統。

Critical Analysis

🔍 來源偏差與資訊缺口

  • 推廣者視角：Spisak 是 AI 自動化社群的經營者，文末直接導流到付費社群。文章傾向呈現功能的理想狀態，而非實際使用的摩擦點。需要打 2-3x 折扣。
  • 未提及的關鍵限制：文章完全沒有討論 Dispatch/Cowork 的安全性模型——誰有權限觸發？檔案存取範圍？操作紀錄是否可審計？對於在企業或合規環境中評估此工具的人，這些是決定性問題。
  • 「AI 作業系統」的誇大風險：文末稱 Claude 從聊天機器人變成「完整的工作作業系統」。這是行銷語言。實際上它仍然受限於本地執行、單一使用者、無雲端冗餘——離真正的「作業系統」差距甚大。
  • Plugin 黑盒問題：「一鍵安裝」的便利性掩蓋了一個重要問題——你無法完整審計 Plugin 的行為。在 GMP 環境中，這等同於 CSV（Computer System Validation）未通過的軟體。
  • 缺少對比基準：文章沒有將此架構與 Make.com、n8n、Zapier 等成熟自動化平台比較。Scheduled Tasks 的「電腦必須開著」限制在這些雲端工具面前是明顯劣勢。

Key Takeaways

1. 五層不是五個產品，是一個系統
Dispatch / Cowork / Plugins / Skills / Scheduled Tasks 是介面層、能力層、編排層的分工。理解它們的互動關係比記住個別功能更重要。
2. Skills 是最大的客製化槓桿
Skills 把 Claude 從通用工具變成你的專屬員工。ZIP 上傳 + 自動偵測的設計讓 Custom Skills 成為整個生態系統中 ROI 最高的投入點。
3. 本地執行是雙面刃
本地執行意味著 Claude 可以存取你的所有檔案（強大），但也意味著電腦必須開著、無雲端冗餘（脆弱）。Mission-critical 排程仍需雲端自動化工具。
4. 合規環境需要額外驗證
文章未討論安全性、審計軌跡、權限控管。在 GMP/合規環境中使用前，需要獨立評估 Plugin 權限範圍、資料存取行為、和操作可追溯性。

Practice Questions

Q1 概念理解 Dispatch 和 Cowork 的關係是什麼？為什麼 Dispatch 不能單獨運作？

Dispatch 是觸發介面（手機端），Cowork 是執行引擎（桌面端）。Dispatch 不能單獨運作，因為所有實際操作（檔案讀寫、外部工具連接、子代理人調度）都發生在 Cowork 模式中。Dispatch 只是把指令從手機同步到桌面端的 Cowork session。沒有 Cowork，Dispatch 就只是一個無法送達的訊息。

Q2 產業應用 如果你要在 CDMO 環境中導入 Claude 的 Plugin 系統，第一個要回答的合規問題是什麼？

第一個問題是：「這個 Plugin 存取了哪些資料，操作紀錄是否可審計？」 在 GMP 環境中，任何接觸產品品質資料的軟體都需要通過 CSV（Computer System Validation）。Plugin 的「一鍵安裝」設計意味著你可能不清楚它存取了哪些檔案、是否修改了資料、操作紀錄是否符合 ALCOA+ 標準。在通過這些驗證之前，Plugin 只能用於非 GMP 的行政/個人任務。

Q3 批判思考 Spisak 稱 Claude 已從聊天機器人變成「完整的工作作業系統」。用你的 2-3x Discount 框架評估這個說法。

打折後的評估：Claude Desktop 生態系統確實在「個人生產力代理」方面有顯著進展，但離「作業系統」還有結構性差距：

缺少的 OS 特性：多使用者支援、雲端冗餘/高可用性、權限管理（RBAC）、完整的審計軌跡、API 可程式化控制（目前主要是 GUI 操作）。

更準確的描述：「個人 AI 助理平台」——強大，但仍是單人單機的工具。Spisak 的 "operating system" 比喻是為了社群行銷的感染力，不是技術架構的準確描述。打 2x 折扣後，它是一個「非常好的個人自動化工具，有潛力在特定場景取代部分重複性工作」。

🎯 我的行動結論
✔ 核心洞察
Anthropic 的產品策略已經從「最好的 LLM」轉向「最好的 AI Agent 平台」。Dispatch + Cowork + Plugins + Skills + Scheduled Tasks 這五層架構是他們對「AI 如何實際完成工作」的系統性回答。我在 Claude Projects 中建立的 Skills 系統（artifact-skill、amaran-presentation 等）正好是這個方向的早期實踐——未來可能直接遷移到 Cowork Skills 體系。
📈 立即應用
本週行動：(1) 在 Claude Desktop 上實測 Cowork 模式，跑一個完整的「搜尋檔案 → 整理 → 輸出報告」流程，感受與 Claude Projects 的差異。(2) 評估現有 Custom Skills（artifact-skill、amaran-presentation）是否可以 ZIP 打包上傳到 Cowork Skills 系統。(3) 不要急著用 Scheduled Tasks——先確認 Make.com 已有的排程是否足夠。
⚠ 需要補充
文章缺少三個關鍵面向：(1) 安全性模型——Plugin/Connector 的權限範圍和審計能力，這決定了能否在 CDMO 工作中使用。(2) 與現有自動化工具（Make.com, n8n）的定位差異——本地 vs 雲端執行的 tradeoff。(3) 成本分析——這些功能需要 Pro/Max 訂閱，但文章沒有比較不同層級的功能差異。需要查 Anthropic 官方文件補充這些缺口。
