Part I: General Controls (基本控制快捷鍵)

Claude Code 的互動模式提供了一組基本控制快捷鍵，讓你在終端機中高效操作。這些快捷鍵涵蓋從中斷生成、清除畫面到切換詳細輸出等常見操作。

"Keyboard shortcuts may vary by platform and terminal. Press ? to see available shortcuts for your environment."

特別值得注意的是 Esc + Esc（雙擊 Escape）可以回溯程式碼和對話，將狀態恢復到之前的某個時間點。這對於 AI 生成了不理想的程式碼時特別有用，類似 Git 的 revert 但作用在對話層級。

"Esc + Esc — Rewind the code/conversation: Restore the code and/or conversation to a previous point."

Tab 鍵用來切換 Extended Thinking（延伸思考），Shift+Tab 則在權限模式之間切換：Auto-Accept Mode（自動接受）、Plan Mode（規劃模式）和一般模式。

"Tab — Toggle extended thinking: Switch between Thinking on and Thinking off."

💡 比喻說明
把 Claude Code 的權限模式想像成製藥廠的品質管控等級：一般模式就像每個步驟都需要 QA 簽核；Plan Mode 像是先審核整個批次計畫再放行；Auto-Accept 則像已驗證的自動化產線，信任 AI 直接執行。根據任務風險等級選擇對應模式。

Part II: Multiline Input & Quick Commands (多行輸入與快捷指令)

Claude Code 支援多種多行輸入方式，這在需要貼上程式碼片段或撰寫複雜提示時非常實用。最通用的方式是在行尾加上 \ 再按 Enter，這在所有終端機都能使用。

"Quick escape: \ + Enter — Works in all terminals."

Quick Commands（快捷指令）是以特殊字元開頭的快速操作。# 開頭可以直接將內容寫入 CLAUDE.md 記憶檔案；/ 是 Slash Command；! 是 Bash 模式；@ 觸發檔案路徑自動補全。

"# at start — Memory shortcut: add to CLAUDE.md. / at start — Slash command. ! at start — Bash mode. @ — File path mention."

📚 核心概念
Quick Commands 速查表：

前綴功能範例
#寫入 CLAUDE.md# Always use TypeScript
/Slash Command/vim、/clear
!直接執行 Bash! npm test
@檔案路徑補全@src/index.ts
🛠️ 實務應用
CDMO 情境：如果你在用 Claude Code 開發 batch record 自動化腳本，# 指令可以讓你持久保存專案規則，例如 # Always validate against 21 CFR Part 11 requirements，確保 AI 在每次對話都記得合規需求。

Part III: Vim Editor Mode (Vim 編輯模式)

Claude Code 內建 Vim 風格編輯模式，可以透過 /vim 指令啟用，或在 /config 中永久設定。對於熟悉 Vim 的開發者來說，這意味著可以在不離開 Claude Code 的情況下，使用熟悉的按鍵操作來編輯輸入。

"Enable vim-style editing with /vim command or configure permanently via /config."

Vim 模式支援標準的 NORMAL / INSERT 模式切換，包含 i、a、o 等進入插入模式的方式，以及 Esc 返回一般模式。導航指令涵蓋 h/j/k/l 方向移動、w/e/b 單字跳轉，以及 gg/G 跳到開頭或結尾。

"Navigation (NORMAL mode): h/j/k/l — Move left/down/up/right. w — Next word. gg — Beginning of input. G — End of input."

編輯指令支援 dd（刪除整行）、dw（刪除單字）、cc（更改整行）等操作，以及 .（重複上次變更）。這些是 Vim 的核心操作子集，足以應付大多數輸入編輯場景。

"Editing (NORMAL mode): dd — Delete line. dw/de/db — Delete word/to end/back. . — Repeat last change."

💡 比喻說明
Vim 模式就像操作無菌隔離器的手套箱：你有兩種狀態 — 觀察模式（NORMAL，手在手套箱外看著裡面）和操作模式（INSERT，手伸進手套箱直接操作）。按 Esc 就是把手抽回來觀察全局，按 i 就是伸手進去精確操作。

Part IV: Command History & Reverse Search (指令歷史與搜尋)

Claude Code 維護基於工作目錄的指令歷史，可以用上下方向鍵瀏覽。使用 /clear 會清除歷史。值得注意的是，歷史展開 (!) 預設是關閉的，以避免和 Bash 模式的 ! 前綴衝突。

"History is stored per working directory. Cleared with /clear command. Note: History expansion (!) is disabled by default."

Ctrl+R 啟動反向搜尋功能，讓你互動式地搜尋先前的指令。輸入關鍵字後，搜尋詞會被高亮顯示在匹配結果中。按 Ctrl+R 繼續往前搜尋更舊的匹配項；按 Tab 或 Esc 接受匹配但繼續編輯；按 Enter 直接執行。

"Press Ctrl+R to interactively search through your command history. Press Ctrl+R again to cycle through older matches."

📚 核心概念
Ctrl+R 搜尋流程：

1. 按 Ctrl+R 啟動搜尋
2. 輸入搜尋關鍵字（搜尋詞會高亮）
3. 按 Ctrl+R 往前翻找更舊的匹配
4. 接受結果：
   - Tab / Esc → 接受並繼續編輯
   - Enter → 接受並立即執行
5. 取消：Ctrl+C 或 Backspace（空搜尋時）
🛠️ 實務應用
在開發 Make.com automation 腳本時，你可能反覆調整同一個提示讓 Claude 修改程式碼。Ctrl+R 可以快速找回之前用過的長提示，避免重新輸入。這比用滑鼠滾動歷史記錄快得多。

Part V: Background Bash Commands & Bash Mode (背景指令與 Bash 模式)

Claude Code 支援背景執行 Bash 指令，讓長時間運行的程序（如 build、test）在背景執行，你可以同時繼續與 Claude 互動。啟動方式有兩種：直接提示 Claude 在背景執行，或在指令執行時按 Ctrl+B 將其移到背景。

"Claude Code supports running bash commands in the background, allowing you to continue working while long-running processes execute."

背景任務有唯一的 ID 用於追蹤，輸出會被緩衝，Claude 可以用 BashOutput 工具取回結果。當 Claude Code 退出時，背景任務會被自動清理。特別注意：tmux 使用者需要按兩次 Ctrl+B，因為 tmux 預設用 Ctrl+B 作為前綴鍵。

"Background tasks have unique IDs for tracking and output retrieval. Background tasks are automatically cleaned up when Claude Code exits."

! 前綴的 Bash 模式可以繞過 Claude，直接在終端執行指令。指令和輸出都會被加入對話上下文，所以 Claude 仍然「知道」你做了什麼。這對於 git status、npm test 等快速操作特別方便。

"Run bash commands directly without going through Claude by prefixing your input with !. Adds the command and its output to the conversation context."

💻 Bash 模式範例
! npm test
! git status
! ls -la
! python validate_batch.py --lot 2025001

這些指令會直接執行，輸出會被加入對話上下文。Claude 不需要解讀或批准指令。

⚠️ 重點提示
Background vs Bash Mode 差異：

背景執行 (Ctrl+B)：長時間運行的程序（build、test runner、dev server），不阻擋你繼續和 Claude 對話。

Bash 模式 (! 前綴)：快速的一次性指令，即時看到結果，並自動加入上下文讓 Claude 知道。

兩者都支援 Ctrl+B 背景化。
💡 比喻說明
背景執行就像在 fill-finish 產線上讓一個環境監測程序持續在背景運行，而你繼續處理其他批次作業。Bash 模式則像是快速查看一下設備面板上的即時數據 — 你不需要透過系統發出正式指令，直接看一眼就好，但這個觀察記錄仍然會被記錄在批次紀錄中。

Key Takeaways

Esc+Esc 是最強保險
雙擊 Escape 可以回溯程式碼和對話到先前狀態，這是 AI 生成程式碼時最重要的安全網。搭配 Checkpointing 功能使用效果更佳。
四個前綴記住就夠
# 記憶、/ 指令、! Bash、@ 檔案 — 這四個快捷前綴涵蓋 80% 的高頻操作，是提升效率的核心。
背景執行釋放對話
Ctrl+B 將長時間運行的 build/test 移到背景，讓你可以同時繼續和 Claude 對話。這將 AI coding 從同步變成非同步工作流。
權限模式匹配風險
Shift+Tab 在三種權限模式間切換。高風險操作用一般模式逐步確認，成熟重複性任務可切到 Auto-Accept 提速。根據任務選擇信任等級。

Practice Questions

Q1 概念理解 在 Claude Code 中，Ctrl+L 和 /clear 有什麼差異？為什麼這個差異在長時間的 coding session 中很重要？

Ctrl+L 只清除終端畫面的視覺內容，但保留完整的對話歷史。Claude 仍然記得之前所有的對話上下文。而 /clear 會清除指令歷史。

這在長時間 coding session 中非常重要，因為保留對話歷史意味著 Claude 仍然理解你的專案上下文（檔案結構、已修改的程式碼、先前的決策），而你只是想要一個乾淨的終端畫面。如果誤用 /clear，Claude 可能會失去重要的上下文，導致需要重新建立專案理解。

Q2 產業應用 假設你正在用 Claude Code 開發一個 batch record 自動化系統，需要同時執行驗證測試和繼續編寫新功能。你會如何組合使用本文介紹的功能來最佳化這個工作流？

建議的工作流組合：

1. 用 # 指令將 GMP 合規規則寫入 CLAUDE.md（例如 # All data must comply with ALCOA+ principles），確保每次對話都自動載入合規上下文。

2. 開發到一個段落後，用 ! pytest tests/ 快速執行測試，輸出會自動加入上下文讓 Claude 知道測試結果。

3. 如果需要執行完整的 integration test（耗時較久），用 Ctrl+B 將其移到背景，然後繼續和 Claude 討論下一個功能模組。

4. 對於驗證性質的程式碼修改，使用一般權限模式逐步確認；對於重複性的樣板程式碼生成，切換到 Auto-Accept 提速。

Q3 批判思考 這份文件是一份 reference documentation。它缺少什麼關鍵資訊？如果你是 Anthropic 的文件團隊，你會建議增加什麼內容？

缺少情境化建議：文件列出了所有快捷鍵，但沒有說明「什麼時候應該用什麼」。新使用者面對一張長表格可能不知道從何開始。建議增加一個「Top 5 shortcuts to learn first」的入門指南。

🎯 我的行動結論
✔ 核心洞察
Claude Code 的互動模式設計哲學是「讓手留在鍵盤上」。四個前綴（# / ! @ ）加上背景執行，本質上是在 AI-assisted coding 中重建了類似 Vim 的效率哲學。對我來說，Esc+Esc 的回溯功能和 Ctrl+B 的背景執行是最有價值的 — 前者是 AI 生成不確定性下的安全網，後者讓 coding session 從同步變成非同步。
📈 立即應用
這週在使用 Claude Code 開發自動化腳本時，刻意練習三個操作：(1) 用 # 將 CDMO 專案的合規規則寫入 CLAUDE.md；(2) 用 ! 前綴直接執行 git 和測試指令；(3) 長時間 build 時用 Ctrl+B 背景化。目標是一週後這三個操作變成肌肉記憶。
