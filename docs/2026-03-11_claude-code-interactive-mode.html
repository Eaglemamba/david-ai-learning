Part I
通用控制與文字編輯快捷鍵
Claude Code 的鍵盤快捷鍵可能因平台和終端機而異。按下 ? 可查看當前環境可用的快捷鍵。
Keyboard shortcuts may vary by platform and terminal. Press ? to see available shortcuts for your environment.
macOS 用戶注意：Option/Alt 鍵快捷鍵（Alt+B、Alt+F 等）需要在終端機中將 Option 設定為 Meta 鍵。iTerm2 在 Settings → Profiles → Keys 中設定；Terminal.app 在 Settings → Profiles → Keyboard 中勾選。
macOS users: Option/Alt key shortcuts (Alt+B, Alt+F, Alt+Y, Alt+M, Alt+P) require configuring Option as Meta in your terminal.
⚠️ 重點提示
最常用的 5 個快捷鍵：
Ctrl+C 取消當前輸入/生成 | Ctrl+L 清屏（保留對話） | Ctrl+B 背景執行任務 | Ctrl+R 反向搜尋歷史 | Esc+Esc 回退/摘要
通用控制涵蓋了從中斷當前操作（Ctrl+C）、終止背景 Agent（Ctrl+F 按兩次確認）、退出會話（Ctrl+D）到切換詳細輸出（Ctrl+O）的基礎操作。
General controls cover canceling current input or generation (Ctrl+C), killing all background agents (Ctrl+F, press twice within 3 seconds to confirm), exiting the session (Ctrl+D), and toggling verbose output (Ctrl+O).
Ctrl+G 可在預設文字編輯器中編輯你的 prompt 或自訂回應 — 適合需要撰寫長篇 prompt 時使用。Ctrl+V（或 macOS iTerm2 的 Cmd+V）可以直接從剪貼簿貼上圖片。
Ctrl+G opens your prompt in the default text editor. Ctrl+V (or Cmd+V in iTerm2) pastes an image from clipboard — it pastes an image or path to an image file.
文字編輯快捷鍵遵循標準 Emacs/readline 風格：Ctrl+K 刪除到行尾、Ctrl+U 刪除整行、Ctrl+Y 貼回刪除的文字。Alt+B/Alt+F 以單詞為單位前後移動游標。
Text editing shortcuts follow standard Emacs/readline style: Ctrl+K deletes to end of line, Ctrl+U deletes entire line, Ctrl+Y pastes deleted text. Alt+B/Alt+F move cursor back/forward one word.
🛠️ 實務應用
CDMO 情境：在 Claude Code 中處理批次記錄審查腳本時，Ctrl+B 可以讓 Python 腳本在背景執行，你可以同時請 Claude 協助分析下一批資料。Ctrl+R 則讓你快速找到之前用過的複雜 git 命令。

Part II
多行輸入、快速指令與主題顯示
多行輸入在不同終端機中有多種方式。最通用的是反斜線 + Enter（\ + Enter），所有終端機都支援。macOS 預設使用 Option+Enter，而 Shift+Enter 在 iTerm2、WezTerm、Ghostty、Kitty 中無需設定即可使用。
Multiline input has multiple methods across terminals. The quick escape method (\ + Enter) works in all terminals. macOS defaults to Option+Enter, and Shift+Enter works out of the box in iTerm2, WezTerm, Ghostty, Kitty.
對於其他終端機（VS Code、Alacritty、Zed、Warp），需要執行 /terminal-setup 來安裝 Shift+Enter 的鍵位綁定。Ctrl+J 是換行字元（Line Feed），也可以用於多行輸入。直接貼上程式碼區塊或 log 時，會自動進入貼上模式。
For other terminals (VS Code, Alacritty, Zed, Warp), run /terminal-setup to install the binding. Ctrl+J is a line feed character for multiline. Paste mode activates when pasting code blocks or logs directly.
快速指令有三種前綴：/ 用於 slash commands 和 skills；! 用於 Bash 模式（直接執行 shell 命令並將結果加入對話脈絡）；@ 觸發檔案路徑自動完成。
Quick commands have three prefixes: / at start for commands or skills; ! at start for Bash mode (run commands directly and add output to session); @ for file path autocomplete.
💡 比喻說明
三種前綴像三條車道：
/ 是「Claude 專用車道」— 啟動 Claude 的內建功能
! 是「系統直通車道」— 繞過 Claude 直接跟系統對話
@ 是「導航輔助車道」— 幫你找到正確的檔案路徑
主題與語法高亮：在 /theme 選擇器內，Ctrl+T 可切換程式碼區塊的語法高亮功能。主題包含亮色和暗色變體、色盲友善（daltonized）主題，以及使用終端機色彩面板的 ANSI 主題。語法高亮僅在 Claude Code 的原生版本中可用。
Inside the /theme picker menu, Ctrl+T toggles syntax highlighting for code blocks. Themes include light and dark variants, colorblind-accessible (daltonized) themes, and ANSI themes that use your terminal's color palette. Syntax highlighting is only available in the native build of Claude Code.

Part IV
Vim 編輯模式
使用 /vim 指令啟用 vim 風格的編輯，或透過 /config 永久設定。支援完整的模式切換（NORMAL / INSERT）、NORMAL 模式下的導航和編輯操作，以及 text objects。
Enable vim-style editing with /vim command or configure permanently via /config. Supports mode switching (NORMAL / INSERT), navigation and editing in NORMAL mode, and text objects.
模式切換：Esc 進入 NORMAL 模式，i/I/a/A/o/O 進入 INSERT 模式（各有不同的插入位置）。導航使用標準 h/j/k/l，以及 w/e/b 單詞移動、0/$/^ 行首行尾、gg/G 文件首尾。
Mode switching: Esc enters NORMAL mode; i/I/a/A/o/O enter INSERT mode at different positions. Navigation uses standard h/j/k/l, w/e/b for word movement, 0/$/^ for line positions, gg/G for document start/end.
Text Objects 與 d、c、y 操作符配合使用：iw/aw（inner/around word）、i"/a"（引號內/含引號）、i(/a(（括號內/含括號）等。還支援字元跳轉 f{char}/t{char} 及重複 ;/,。
Text objects work with operators like d, c, and y: iw/aw (inner/around word), i"/a" (inner/around double quotes), i(/a( (inner/around parentheses). Also supports character jumps f{char}/t{char} and repeat ;/,.
💡 比喻說明
Vim 模式的意義不在 Vim 本身。它代表 Claude Code 的設計哲學：不要求用戶改變既有的肌肉記憶，而是讓工具適應用戶。就像 GMP 系統中好的 MES 應該適配既有流程而非反過來。
特殊行為：在 vim NORMAL 模式下，如果游標在輸入的開頭或結尾且無法繼續移動，方向鍵會改為瀏覽命令歷史。這個細節體現了「不同情境下相同按鍵有不同語義」的 vim 哲學。
In vim normal mode, if the cursor is at the beginning or end of input and cannot move further, the arrow keys navigate command history instead.

Part V
背景任務與命令歷史
Claude Code 支援在背景執行 bash 指令，讓你在長時間程序執行期間繼續工作。背景執行後立即返回一個 task ID，Claude Code 可以在背景指令執行期間回應新的 prompt。
Claude Code supports running bash commands in the background, allowing you to continue working while long-running processes execute. It immediately returns a background task ID. Claude Code can respond to new prompts while the command continues executing.
啟動背景執行有兩種方式：(1) 直接請 Claude Code 在背景執行指令；(2) 按 Ctrl+B 將正在執行的 Bash 工具調用移到背景。Tmux 用戶需要按兩次 Ctrl+B，因為 tmux 的前綴鍵也是 Ctrl+B。
You can either prompt Claude Code to run a command in the background, or press Ctrl+B to move a regular Bash tool invocation to the background. Tmux users must press Ctrl+B twice due to tmux's prefix key.
🛠️ 實務應用
常見的背景指令場景：Build tools（webpack, vite, make）、Package managers（npm, yarn）、Test runners（jest, pytest）、Development servers、Long-running processes（docker, terraform）。

CDMO 應用：在 Claude Code 中執行 Python 批次記錄分析腳本時，用 Ctrl+B 把它放到背景，同時繼續請 Claude 協助撰寫下一個分析模組。這把線性工作流轉變為並行。
輸出緩衝：背景任務的輸出會被緩衝，Claude 可以使用 TaskOutput 工具取回。背景任務有唯一 ID 用於追蹤。當 Claude Code 退出時，背景任務會自動清理。設定環境變數 CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1 可禁用所有背景任務功能。
Output is buffered and Claude can retrieve it using the TaskOutput tool. Background tasks have unique IDs for tracking. Background tasks are automatically cleaned up when Claude Code exits. Set CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1 to disable.
命令歷史功能：輸入歷史按工作目錄儲存，執行 /clear 時會重置（但之前的會話可以用 /resume 恢復）。使用上下箭頭瀏覽歷史。Ctrl+R 啟動反向搜尋：輸入文字搜尋之前的指令，Ctrl+R 循環匹配結果，Tab/Esc 接受匹配，Enter 接受並執行。
Input history is stored per working directory and resets when you run /clear. Use Up/Down arrows to navigate. Ctrl+R activates reverse search: type to search, Ctrl+R to cycle matches, Tab/Esc to accept, Enter to accept and execute. History expansion (!) is disabled by default.

Part VI
側問、提示建議、任務清單與 PR 狀態
/btw 讓你在不影響對話歷史的情況下快速問一個問題。它能看到當前對話的完整脈絡，但沒有工具存取權限（不能讀檔、執行指令或搜尋）。回答以浮動視窗顯示，按 Space/Enter/Esc 關閉。
Use /btw to ask a quick question about your current work without adding to the conversation history. It has full visibility into the current conversation but no tool access — Claude cannot read files, run commands, or search. Press Space, Enter, or Escape to dismiss.

提示建議（Prompt Suggestions）：開啟會話時，會出現灰色的範例指令建議，基於你專案的 git 歷史。Claude 回應後，建議會基於對話歷史持續出現。按 Tab 接受建議，按 Enter 接受並送出。建議以背景請求執行，重用 prompt cache，額外成本極低。
A grayed-out example command appears based on your project's git history. After Claude responds, suggestions continue based on conversation history. Press Tab to accept, Enter to accept and submit. The suggestion runs as a background request reusing the parent conversation's prompt cache, so additional cost is minimal.
Task List（任務清單）：處理複雜多步驟工作時，Claude 會建立任務清單追蹤進度。按 Ctrl+T 切換顯示（最多 10 項）。任務在 context 壓縮後仍然保留，幫助 Claude 在大型專案中保持組織。設定 CLAUDE_CODE_TASK_LIST_ID=my-project 可在會話間共享任務清單。
When working on complex multi-step work, Claude creates a task list to track progress. Press Ctrl+T to toggle (shows up to 10 tasks). Tasks persist across context compactions. Set CLAUDE_CODE_TASK_LIST_ID=my-project to share task lists across sessions.
PR 審查狀態：當你在有開啟 PR 的分支上工作時，Claude Code 在 footer 顯示可點擊的 PR 連結，底線顏色表示審查狀態：綠色=已批准、黃色=待審查、紅色=要求修改、灰色=草稿、紫色=已合併。需要安裝 gh CLI 並認證。
When working on a branch with an open PR, Claude Code displays a clickable PR link in the footer. The colored underline indicates review state: green=approved, yellow=pending, red=changes requested, gray=draft, purple=merged. Requires gh CLI to be installed and authenticated.

Practice Questions

Q1 概念理解 /btw 和 Subagent 的設計哲學差異是什麼？在什麼情境下你會選擇 /btw 而非 Subagent？

/btw 是「用已知脈絡回答問題」，Subagent 是「去發現新資訊」。

/btw 擁有完整的對話脈絡但沒有工具存取權限（不能讀檔、執行指令），成本低因為重用 prompt cache。Subagent 有完整工具但從空白脈絡開始，成本較高。

選擇 /btw 的情境：Claude 已經讀過你的程式碼，你想問「剛才那個 config 檔叫什麼名字？」。選擇 Subagent 的情境：你需要 Claude 去搜尋文件或讀取新檔案來回答問題。

CDMO 類比：/btw 像是在批次記錄審查會議中快速問旁邊的同事一個你們都已知道的問題；Subagent 像是派人去查閱 SOP 原文再回報。

Q2 產業應用 在 CDMO 的技術轉移專案中，如何利用 Claude Code 的背景任務（Ctrl+B）和 Task List 功能來管理多個並行的自動化腳本？

核心策略：把線性工作流轉為並行。

假設技術轉移需要同時處理：(1) 解析批次記錄 PDF 提取關鍵參數、(2) 比對不同廠區的 SOP 差異、(3) 生成 gap analysis 報告。

傳統方式是逐一執行，每次等待完成才開始下一個。使用 Claude Code 背景任務：用 Ctrl+B 把 PDF 解析腳本放到背景 → 請 Claude 協助撰寫 SOP 比對邏輯 → 用 TaskOutput 工具取回 PDF 解析結果 → 整合所有結果生成報告。Task List（Ctrl+T）追蹤每個子任務的進度。

關鍵注意：設定 CLAUDE_CODE_TASK_LIST_ID 可以在不同 Claude Code 會話間共享任務清單，適合跨天的大型專案。

Q3 批判思考 Claude Code 的 50+ slash commands 和 Vim 模式反映了什麼樣的使用者假設？這對非技術人員的 AI 採用有什麼啟示？

使用者假設：目標用戶是重度終端機用戶（開發者、DevOps 工程師）。

證據：(1) Vim 模式含 text objects 支援，這只有 vim 老手才會用；(2) 50+ 指令中包含 /diff、/security-review、/pr-comments 等 git-native 操作；(3) 背景任務設計假設用戶知道什麼是 tmux 前綴鍵。

對非技術人員的啟示：Claude Code 不是「讓非工程師也能寫程式」的工具 — 那是 Claude.ai 的 Artifacts 和 Cowork 的角色。Claude Code 是「讓好的工程師變得更快」的工具。在 CDMO 中推動 AI 採用時，需要區分這兩條路線：Claude.ai 適合品質管理、法規事務人員；Claude Code 適合 IT 自動化、數據工程團隊。

文件盲點：這份參考手冊完全缺少「學習路徑」建議 — 新用戶應該先掌握哪 5 個指令？什麼順序學習最有效率？這是操作手冊的典型弱點。

🎯 我的行動結論
✔ 核心洞察
Claude Code 的互動設計反映了一個關鍵決定：讓 AI 助手融入開發者既有的終端機工作流，而非要求開發者適應新的 GUI。50+ slash commands、Vim 模式、背景任務等設計，都是在說「你不需要離開終端機」。對 CDMO 情境，這代表 Claude Code 適合整合進已有 CLI 工具鏈的技術團隊，但對非技術人員的採用門檻較高。
📈 立即應用
本週在 Claude Code 中練習三個快捷鍵組合：(1) Ctrl+B 背景執行 build/test 指令；(2) /btw 在工作中快速查詢而不中斷脈絡；(3) Esc+Esc 用 rewind 功能恢復到之前的狀態。同時在 CLAUDE.md 中記錄常用的自訂 skills。
⚠ 需要補充
文件是純參考手冊，缺少「常見工作流範例」和「效率提升量化數據」。例如：背景任務搭配 Task List 的典型工作流是什麼？使用快捷鍵 vs. 手動操作的時間差距多少？這些需要自己在實際使用中累積經驗。另外，部分指令（如 /chrome、/desktop）暗示 Claude Code 正在擴展到 CLI 以外的平台，值得持續關注。
