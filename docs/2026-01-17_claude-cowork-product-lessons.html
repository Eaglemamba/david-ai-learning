Part I: Claude Cowork - 產品策略的經典案例

Claude 發布了名為 Cowork 的新產品，它將 Claude Code 中已有的許多功能重新定位到工作情境中。 "Claude revealed a new product called Cowork which takes many of the features already available in Claude Code and repositions them for a work context."

Anthropic 概述的使用案例包括：建立檔案、處理數據、製作原型和整理檔案。例如，你可以讓 Claude 存取電腦上的資料夾，然後它會將檔案整理成試算表。 "Some of the use cases outlined by Anthropic in Cowork include things like creating files, crunching data, making prototypes and organizing files."

這些功能在 Claude 中本來就有，但透過將產品重新包裝成一個住在終端機中、較不令人畏懼的工具，Claude Cowork 希望能在 12 月達成的 10 億美元年收入基礎上繼續成長。 "These use cases were already available in Claude but by repackaging the offering into a less intimidating tool that lives in the terminal, Claude Cowork is hoping that it can build upon the $1 billion dollar revenue run rate that it achieved in December."

Core Concept: Product Spin-off Strategy

什麼是 Product Spin-off？

當一個產品的某個用戶群展現出非預期的使用模式時，將該功能獨立出來，針對新用戶群重新包裝。Cowork 就是從 Claude Code 「spin-off」出來的子產品。

關鍵成功因素：

  • 降低進入門檻（less intimidating）
  • 針對新用戶群調整 UX
  • 保留核心技術能力
Analogy: CDMO 的產線分拆

這就像 CDMO 發現某條多功能產線被特定客戶用於固定流程，於是獨立出一條專用產線，降低換線時間、優化該客群體驗。技術能力相同，但包裝和服務模式針對特定需求優化。

這個新產品是 Anthropic 在 2026 年頂尖產品團隊所做的事情的絕佳範例。他們發布了一個產品（Claude Code），意識到非工程師也從中獲得價值，並利用這些數據來啟發一個全新的實驗性子產品 Claude Cowork。 "The new product is a fine example of Anthropic doing what top product teams do in 2026. They shipped a product (Claude Code), realised that non-engineers found value in it and used that data to inform an entirely experimental new sub-product with Claude Cowork."

這個產品僅用 10 天就完成開發，最終可能會失敗，但 AI 正在讓這類快速產品實驗變得更容易執行。 "It was apparently hacked together in just 10 days and may ultimately fail, but AI is making these types of quickfire product experiments much easier to run."

Key Insight: 10 天產品實驗

這代表什麼？

AI 工具正在壓縮產品開發週期。過去需要數月的產品原型，現在可以在 10 天內完成並推向市場測試。

對產品團隊的影響：

  • 實驗成本大幅降低
  • 失敗的代價變小，可以更大膽
  • 「Ship first, learn later」成為可行策略
Part II: Claude Code 的非工程應用場景

雖然 Cowork 目前僅對 Max 訂閱用戶開放，但 Claude Code 已經能很好地幫助非工程師處理工作。以下是一些你可以探索的使用案例： "Cowork is currently only available for users on the Max tier but Claude Code already does an excellent job of helping non-engineers at work."

PRD 和 Jira Ticket 撰寫 Agent - 建立自訂 AI agent，使用預設模板和 /agent 指令自動生成產品需求文件和 Jira tickets。 "PRD and Jira Ticket Writing Agents - create custom AI agents that automatically generate product requirement documents and Jira tickets using pre-configured templates and the /agent command"

SEO 稽核 - 使用 Claude Code 搭配 Playwright 造訪網站、分析 SEO 效能、進行關鍵字研究，甚至根據研究結果建立登陸頁面。 "SEO Audits - use Claude Code with Playwright to visit websites, analyze SEO performance, conduct keyword research, and even create landing pages based on that research"

檔案整理和編輯 - 使用自然語言指令整理本地檔案（例如按日期排序截圖）、編輯試算表、轉換檔案格式（CSV 轉 JSON）、批次編輯文件。 "File organization and editing - organize local files using natural language commands (e.g., sorting screenshots by date), edit spreadsheets, convert file formats (CSV to JSON), and bulk edit documents"

第二大腦/知識管理 - 建立個人知識系統，維護「brain dump」資料夾，讓 Claude Code 分析你的筆記、建立心智圖、識別模式，並將想法整理成類別。 "Second Brain/Knowledge Management - create a personal knowledge system by maintaining 'brain dump' folders where Claude Code analyzes your notes, creates mind maps, identifies patterns, and organizes thoughts into categories"

Practical Application: CDMO 可用場景

對於 CDMO Operations Director：

  • Batch Record 分析 - 批次處理多份 batch records，提取關鍵數據點
  • 偏差報告整理 - 自動分類和彙整偏差報告，識別趨勢
  • SOP 版本比對 - 比較不同版本 SOP 的變更內容
  • 供應商文件管理 - 整理和索引供應商提交的文件

注意：這些應用需要考慮 21 CFR Part 11 合規性和數據安全要求。

Part III: Google 的反擊 - Antigravity 與 Personal Intelligence

Google 對 Claude Code 的回應 - Antigravity - 現在正式支援之前稱為 Claude Skills（現已更名為 Agent Skills，因為它們是開放標準）的功能。 "Google's answer to Claude Code, Antigravity, now officially supports what was previously called Claude Skills (now renamed to Agent Skills since they're an open standard)."

Google 本週還揭示了 Gemini 的新功能 - Personal Intelligence。令人驚訝的是 Google 花了這麼長時間才發布這樣的產品，但 Personal Intelligence 允許用戶連接 Gmail、Photos、YouTube 和 Search 等應用程式。 "Google also revealed a new feature for Gemini this week called 'Personal Intelligence'... Personal Intelligence allows users to connect apps like Gmail, Photos, YouTube and Search."

隨著 context 對 AI 助理變得越來越重要，Google 坐擁的數據護城河是 ChatGPT、Perplexity 等競爭對手根本無法匹敵的。雖然這些新功能是 opt-in only，但從某種意義上說，這些工具可能會讓 Gemini「過於強大」。 "As context becomes increasingly important for AI Assistants, Google is sitting on a data moat that ChatGPT, Perplexity and others simply cannot compete with."

目前這項功能僅在美國提供，根據歐盟的數位市場法 (DMA)，Google 被指定為「守門人」(gatekeeper)，不能優待自家服務，這可能意味著它在其他地區的推出會延遲。 "For now, this is available in the US only and under the EU's DMA, Google is a designated 'gatekeeper' and cannot prefer its own services which could mean it is delayed before rolling out elsewhere."

Core Concept: Data Moat

什麼是 Data Moat（數據護城河）？

指一家公司因為擁有競爭對手無法複製的數據資產，而形成的競爭優勢。Google 的護城河包括：

  • Gmail - 你的通訊歷史和聯絡人
  • Photos - 你的視覺記憶和地點
  • YouTube - 你的興趣和觀看習慣
  • Search - 你的好奇心和需求

這讓 Gemini 可以提供真正「了解你」的個人化服務，而其他 AI 只能從頭開始了解每位用戶。

Regulatory Watch: EU DMA

Digital Markets Act (DMA) 是歐盟的反壟斷法規，要求被指定為「守門人」的大型平台不能優待自家服務。

這意味著 Google 在歐盟不能強制 Personal Intelligence 使用 Gmail 而非其他郵件服務，這可能會延遲或改變產品在歐洲的推出方式。

Part IV: Replit Mobile Apps - 從原型到 App Store

Replit 推出了 Replit Mobile Apps - 一個快速建立端對端行動應用的新工具。它將前端整合到資料庫、身份驗證和第三方 API（如 OpenAI、Apple Maps、Twilio 等）。當 App 準備好時，可以直接推送到 App Store。 "Replit has launched Replit Mobile Apps - a new tool for building end to end mobile apps quickly. It integrates the front end to a database, authentication and third party apps APIs like OpenAI, Apple Maps, Twilio and more. When an App is ready, it can be pushed to the App Store."

Replit 將這些應用定位為「production-ready」而非原型。然而，可以說「vibe-coded」的行動應用不太可能滿足更大、更複雜的產品組織的需求。 "Replit is positioning these apps as production-ready rather than prototypes. Still, it's arguable that vibe-coded mobile apps are unlikely to meet the needs of larger, more complex product organizations."

但對於還沒有行動應用的小型團隊來說，這看起來是一個快速將原型或實驗性應用交到用戶手中的好方法。正如大多數行動產品團隊所知，真正的工作往往是在應用發布到生產環境之後才開始。 "For smaller teams that don't yet have a mobile app, though, this looks like a neat way to get prototypes or experimental apps into users' hands quickly. As most mobile product teams will attest, the real work often begins once an app ships to production."

Analogy: MVP vs Production in Pharma

這就像製藥業的差異：

  • Lab Scale = Vibe-coded app（概念驗證）
  • Pilot Scale = Production-ready（小規模生產）
  • Commercial Scale = Enterprise-grade（大規模、合規、可維護）

Replit 讓你快速到達 Pilot Scale，但 Commercial Scale 需要的穩定性、安全性、可維護性仍然需要傳統工程投入。

Use Case: Internal Tools

適合使用 Replit Mobile Apps 的場景：

  • 內部工具 App（不需要 App Store 審核）
  • 活動或展會專用 App
  • 快速 MVP 驗證用戶需求
  • 簡單的數據收集工具

不適合的場景：需要高安全性、合規審計、複雜業務邏輯的企業應用。

Part V: 市場數據解讀 - 下載量下降但收入上升

2025 年全球行動應用消費者支出達到 1,558 億美元，比 2024 年增長 21.6%。但下載量下降了 2.7%，降至 1,069 億次，低於 2024 年的 1,098 億次和 2023 年的 1,136 億次。 "Global consumer spending on mobile apps reached $155.8 billion in 2025, up 21.6% from 2024. But, downloads decreased by 2.7% to 106.9 billion, down from 109.8 billion in 2024 and 113.6 billion in 2023."

下載量下降但收入上升 21.6%，這些數據表明獲客量變得不如變現效率重要。 "With downloads declining but revenue up 21.6%, the data suggests that acquisition volume is becoming less important than monetization effectiveness."

非遊戲類支出增長了 33.9%，現在已經超過遊戲收入。對於生產力、健康、金融、娛樂或其他非遊戲類別的產品團隊來說，這驗證了用戶願意為實用性和持續價值付出可觀費用。 "Non-game spending grew 33.9% and now exceeds gaming revenue. For product teams in productivity, health, finance, entertainment, or other non-game categories, this validates that users are willing to pay significantly for utility and ongoing value."

Key Insight: Quality over Quantity

這個趨勢反轉意味著什麼？

  • 市場成熟 - 用戶不再隨意下載，而是更謹慎選擇
  • 付費意願提高 - 願意為高品質服務付費
  • 策略轉變 - 從「衝下載量」轉向「提高 LTV」

對產品策略的啟示：專注於 retention 和 monetization，而非 acquisition。

Non-Gaming > Gaming

非遊戲類應用收入首次超越遊戲，這是一個里程碑。代表用戶願意為「工具」和「生產力」付費，而不只是「娛樂」。

成長最快的類別：生產力、健康、金融、娛樂串流。

Part VI: 企業 AI 採用現況 - HBR 調查數據

Harvard Business Review 發布的新調查發現，99% 的領導者表示數據和 AI 投資是首要優先事項，38% 的公司確認已任命 Chief AI Officer。 "A new survey published in Harvard Business Review found that 99% of leaders say that investments in data and AI are a top priority with 38% of companies confirming that they've appointed a Chief AI Officer."

其他可能對產品團隊有價值的數據包括：

  • 39% 報告 AI 已規模化投入生產，比兩年前的 5% 大幅上升
  • 54% 已在有限範圍內將 AI 投入生產，比兩年前的 24% 上升
  • 94% 已超越純實驗階段，比兩年前的 29% 上升

Key Takeaways

1. Ship-Observe-Spinoff

Anthropic 的 10 天產品實驗證明：先發布、觀察用戶行為、再針對新發現的用戶群快速衍生子產品。AI 讓這種實驗成本大幅降低。

2. Data Moat 決勝負

Google 的 Personal Intelligence 展示了數據護城河的威力。在 AI 助理競爭中，誰擁有更多用戶 context，誰就能提供更個人化的服務。

3. 變現 > 獲客

App 下載量下降但收入上升 21.6%。市場成熟後，產品策略應從「衝量」轉向「提高單位價值」和「增強留存」。

4. 94% 已超越實驗

企業 AI 採用已從「要不要做」轉變為「如何做得更好」。還在觀望的組織正在快速落後。

Practice Questions

Q1 概念理解 為什麼 Anthropic 選擇將 Cowork 定位為「less intimidating」的工具？這反映了什麼產品策略思維？

核心洞察：技術能力相同，但用戶體驗和心理門檻是不同的產品維度。

Claude Code 原本是為工程師設計的，使用 terminal 介面對非工程師來說有心理障礙。Cowork 的策略是：

  • 保留核心技術能力（存取檔案系統、處理數據）
  • 重新包裝 UX，降低進入門檻
  • 針對「知識工作者」而非「工程師」設計溝通方式

這反映了「同一技術，不同市場」的產品策略。當發現非目標用戶也在使用產品時，與其改變原產品，不如衍生一個針對新用戶群優化的子產品。

Q2 產業應用 作為 CDMO 的 Operations Director，你如何應用「Ship-Observe-Spinoff」框架來優化內部流程？

框架轉譯到 CDMO 情境：

Ship First：先在一條產線或一個部門試行新的 AI 工具或流程（例如用 Claude 分析 batch records）。不需要完美，只需要可用。

Observe Users：觀察誰在使用、如何使用、有哪些非預期的應用方式。例如，QA 團隊可能發現這個工具對偏差趨勢分析特別有用，即使原本是為 batch release 設計的。

Spinoff：根據觀察，為特定使用案例建立專門的 workflow 或 prompt template。例如，從「通用 batch record 分析」衍生出「偏差模式識別專用工具」。

關鍵考量：在製藥環境中，任何新工具都需要考慮驗證和合規要求。建議從非 GxP 流程開始實驗，累積經驗後再擴展到受監管流程。

Q3 批判思考 Google 的 Personal Intelligence 被描述為「可能過於強大」。你認為這種「數據護城河」策略有什麼潛在風險？

潛在風險多面向：

監管風險：如文章所述，EU DMA 已將 Google 列為「守門人」，不能優待自家服務。這種深度整合可能在更多地區引發反壟斷關注，導致產品功能受限或被迫拆分。

隱私風險：整合越多個人數據，安全漏洞的影響就越大。一次資料外洩可能影響用戶的郵件、照片、搜尋歷史、觀看記錄全部。這會影響用戶信任和採用意願。

競爭風險：過度依賴數據護城河可能導致創新惰性。當護城河是你的主要優勢時，可能會忽視產品本身的創新。OpenAI 和 Anthropic 被迫在模型能力上創新，長期可能建立不同類型的競爭優勢。

用戶鎖定風險：當用戶的所有數據都在 Google 生態系統中，離開成本極高，這可能引發用戶的警覺和反彈。

我的行動結論
核心洞察
10 天產品實驗的可行性改變了產品開發的風險計算。對於我準備 COO 的職涯路徑，這意味著「快速實驗、數據驅動決策」不只是 tech 公司的專利，而是任何組織都應該培養的能力。在 CDMO 環境中，這可以從非 GxP 流程開始應用。
立即應用
本週嘗試用 Claude Code/Cowork 處理一個具體的檔案整理任務（如供應商文件分類或會議記錄彙整），體驗「10 天實驗」的思維。即使只是小規模測試，也能累積判斷「哪些流程適合 AI 輔助」的直覺。
需要補充
文章沒有深入討論 Cowork 與 Claude Code 的具體差異（UX 上究竟改了什麼）。另外，對於企業 AI 採用的 HBR 數據，需要進一步了解「規模化生產」的具體定義和案例，才能對標 CDMO 的成熟度。
