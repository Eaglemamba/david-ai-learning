Part 1: Tier 5 Must-Know Features 必學功能 6 items
#1 of 110+
Security Fix - Sensitive Data Protection
Fixed security issue where sensitive data (OAuth tokens, API keys, passwords) could be exposed in debug logs.
★★★★★
安全修復 - 敏感資料保護

為何這是最高優先級？

API keys 和 OAuth tokens 是存取敏感系統的憑證。現在不再被記錄到 debug logs，符合：

  • 21 CFR Part 11 存取控制要求
  • GDPR 資料保護原則
  • SOC 2 合規要求
#2 of 110+
Language Settings
Added language setting to configure Claude's response language (e.g., language: "japanese").
★★★★★
語言設置

可以讓 Claude 用日語、中文等其他語言回覆。

設定方式：在 settings.json 加入：

language: "traditional-chinese"

無需每次都在 prompt 中指定語言。

#3 of 110+
Bash Tool Permission Wildcards
Added wildcard pattern matching for Bash tool permissions using * at any position in rules (e.g., Bash(npm *), Bash(* install), Bash(git * main)).
★★★★★
Bash 權限通配符

比喻：以前每個工具都要簽名批准。現在可以設定：「所有 npm 開頭的工具都可以直接用」。

  • Bash(npm *) — 所有 npm 命令
  • Bash(git * main) — 對 main 的 git 操作
  • Bash(python *.py) — 執行任何 Python 腳本
#4 of 110+
Output Limit Auto-Continue
Improved Claude to automatically continue when response is cut off due to output token limit, instead of showing an error message.
★★★★★
輸出限制自動繼續

解決的痛點：處理長篇內容時（如分析大型 batch records），輸出被截斷需要手動輸入「continue」。

現在：Claude Code 會自動檢測截斷並無縫續寫，你看到的就是完整回應。

#5 of 110+
Performance Optimizations
Multiple optimizations to improve startup performance. Improved terminal rendering performance when using native installer or Bun, especially for text with emoji, ANSI codes, and Unicode characters. Improved performance when reading Jupyter notebooks with many cells.
★★★★★
效能優化

三個層面的優化：

  • 啟動時間：冷啟動更快
  • 終端渲染：emoji、ANSI 代碼更流暢
  • Jupyter：大型 notebook 不再卡頓

每天啟動數十次，每次省 1-2 秒，累積可觀。

#6 of 110+
Piped Input Reliability
Improved reliability for piped input like cat refactor.md | claude.
★★★★★
管道輸入可靠性

自動化應用：

  • cat instructions.md | claude
  • git diff | claude "review this"

可以將 batch records 透過管道傳給 Claude Code 進行分析。

Part 2: Tier 4 High-Value Features 高價值功能 27 items
#7 of 110+
Skills Hot Reload
Added automatic skill hot-reload - skills created or modified in ~/.claude/skills or .claude/skills are now immediately available without restarting the session.
★★★★☆
技能熱重載

以前：修改 Skill → 重啟 → 測試 → 重啟...

現在：修改 Skill → 立即測試 → 修改 → 立即測試

類比：從「每次都要重新編譯」進化到「Hot Module Replacement」。迭代速度提升 3-5 倍。

#8 of 110+
Skills Isolated Environment
Added support for running skills and slash commands in a forked sub-agent context using context: fork in skill frontmatter.
★★★★☆
技能隔離環境

比喻：QC 實驗室的隔離室

  • Normal：所有測試在同一工作台
  • Fork：獨立隔離室，出錯不污染其他測試

適合測試可能有副作用的自動化腳本。

#9 of 110+
Terminal Shortcuts (Shift+Enter)
Changed Shift+Enter to work out of the box in iTerm2, WezTerm, Ghostty, and Kitty without modifying terminal configs.
★★★★☆
終端快捷鍵改進

輸入多行命令時，Shift+Enter 換行不再需要手動配置終端。

支援：iTerm2、WezTerm、Ghostty、Kitty

新手友好，減少配置問題。

#10 of 110+
Demo Mode (Hide Account Info)
Added CLAUDE_CODE_HIDE_ACCOUNT_INFO environment variable to hide email and organization from the UI, useful for streaming or recording sessions.
★★★★☆
展示模式

使用場景：

  • 錄製教學影片時隱藏 email
  • 直播時保護隱私
  • 截圖分享不洩露組織資訊

export CLAUDE_CODE_HIDE_ACCOUNT_INFO=1

#11 of 110+
Session Resume Fixes
Fixed files and skills not being properly discovered when resuming sessions with -c or --resume. Fixed pasted content being lost when replaying prompts from history. Fixed Esc key behavior with queued prompts.
★★★★☆
會話恢復修復

三個修復點：

  • Files/Skills 路徑解析正確
  • 歷史重放時貼上內容不遺失
  • Esc 不會意外取消執行中任務

長期專案會話恢復更穩定。

#12 of 110+
Permission Prompt Optimization
Reduced permission prompts for complex bash commands. Improved permission prompt UX with Tab hint moved to footer, cleaner Yes/No input labels with contextual placeholders.
★★★★☆
權限提示優化

兩個改進：

  • 複雜命令的權限提示減少
  • Tab 提示移到頁尾，Yes/No 標籤更清晰

減少「權限疲勞」(permission fatigue)。

#13 of 110+
MCP Dynamic Updates (list_changed)
Added support for MCP list_changed notifications, allowing MCP servers to dynamically update their available tools, prompts, and resources without requiring reconnection.
★★★★☆
MCP 動態更新

MCP (Model Context Protocol) 是連接 AI 與外部服務的協議。

現在 MCP 伺服器可以動態新增或移除工具，不需要斷開重連。

適合整合不斷演進的企業系統。

#14 of 110+
Unified Ctrl+B Backgrounding
Added unified Ctrl+B backgrounding for both bash commands and agents - pressing Ctrl+B now backgrounds all running foreground tasks simultaneously.
★★★★☆
統一後台任務快捷鍵

以前 bash 和 agent 有不同的後台化方式。

現在：按 Ctrl+B，當前任務會移到後台繼續執行。

不論是 bash 還是 agent，都用同一個快捷鍵。

#15 of 110+
Hooks for Agents & Skills
Added hooks support to agent frontmatter, allowing agents to define PreToolUse, PostToolUse, and Stop hooks scoped to the agent's lifecycle. Added hooks support for skill and slash command frontmatter.
★★★★☆
Agent 和技能的 Hook 支援

Hook 類型：

  • PreToolUse：工具執行前觸發
  • PostToolUse：執行後觸發（審計記錄）
  • Stop：會話結束時觸發

可用 PostToolUse 自動記錄操作，滿足 ALCOA 可追溯性。

#16 of 110+
Skills Frontmatter Enhancements
Added support for agent field in skills. Added support for YAML-style lists in frontmatter allowed-tools. Added once: true config for hooks. Skills from /skills/ directories now visible in slash command menu by default.
★★★★☆
技能 Frontmatter 增強
  • agent 欄位：指定執行 agent

  • 預設可見：/skills/ 自動出現在選單
#17 of 110+

★★★★☆

Q1理解應用Claude Code 2.1.0 的 Skills 系統新增了context: fork功能。請解釋這個功能的作用。

答案：context: fork 讓 Skill 在隔離的子環境中執行，不會影響主工作區。適用於實驗性操作、多版本測試、風險隔離。

Q2理解應用你會強調哪三個功能來說服 IT 和 QA 主管這個工具是安全且合規的？

1. Security Fix：敏感資料不再記錄到 logs
2. Hook System：自動記錄操作滿足 ALCOA
3. Tool Restriction：精確控制工具範圍

Q3理解應用110+ 項改進大多是 bug fixes 和 UX 改善，從產品策略角度這代表什麼？

產品成熟度：從「早期採用者工具」轉變為「主流生產力工具」
平台化：Skills 強化顯示正在打造可擴展平台
企業準備：安全/權限改進是企業採用的先決條件
