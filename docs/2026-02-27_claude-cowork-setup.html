Part I: Cowork 不是聊天機器人 (Cowork Is Not a Chatbot)
Part I

從對話介面到桌面協作者

大多數人以為 Claude 就像 ChatGPT：一個文字框，你輸入，它回應。那是 claude.ai 的瀏覽器版本。Cowork 完全不同。它住在你的桌面上，能讀寫你電腦上的資料夾檔案，建立 Word 文件、帶有公式的試算表，還能安裝針對你特定工作的專業外掛。
"Most people think Claude is like ChatGPT. A text box. You type, it responds. That's claude.ai. Claude Cowork is different. It lives on your desktop. It reads and writes to folders on your actual computer."
Cowork 的誕生背景是 Anthropic 將 Claude Code（開發者用的 agentic 工具）重新打造給一般知識工作者使用。不需要終端機、命令列或程式碼。你描述想要的結果，指向你的檔案，然後離開。
"Cowork is what happened when Anthropic took the power of Claude Code — their agentic developer tool — and rebuilt it for the rest of us. No terminal. No command line. No code."
Nav Toor 描述了他的典型工作循環：描述需求 → Cowork 問三個問題 → 回答問題 → 20 分鐘後回來看完成的文件。這個循環現在涵蓋他大約 60% 的知識工作。
"I describe what I need. Cowork asks me three questions. I answer them. I come back 20 minutes later to a finished document. That loop now covers about 60% of my knowledge work."
💡 比喻說明
想像你有一位超級能幹的助理。以前（ChatGPT 模式），你必須把文件列印出來遞給他，等他看完再口述回覆，你再手動記錄。現在（Cowork 模式），這位助理坐在你辦公桌旁邊，可以直接打開你的檔案櫃、直接在你的電腦上建立文件、做完放回原位。差別不是「更聰明」，而是「更無摩擦」。
⚠️ 重點提示
Cowork vs claude.ai 的核心差異：
claude.ai = 瀏覽器內的對話介面（上傳/下載循環）
Cowork = 桌面應用程式（直接讀寫本地檔案系統）

這個差異看似微小，但消除了整個「匯出 → 拖入 → 等待 → 下載 → 放回」的循環，從根本上改變了 AI 從「你去找的工具」變成「在你環境中工作的協作者」。
📚 核心概念
Claude Code → Cowork 的演化路線
Claude Code 是面向開發者的命令列 agentic 工具。Cowork 將同樣的「代理式執行」能力包裝成圖形介面，讓非技術使用者也能操作。這代表 Anthropic 的產品策略：先在開發者市場驗證 agent 架構，再推向更廣泛的知識工作者。
Part II: File System Access - 檔案系統存取
Part II - Feature #1

從上傳/下載到直接讀寫

其他所有 AI 工具都靠上傳運作。你匯出檔案、拖進聊天、等待、取得輸出、下載、放回原處。Cowork 消除了整個循環。你選擇電腦上的資料夾，Claude 讀取裡面的所有內容。當它建立東西時，直接存到那個資料夾。
"Every other AI tool runs on uploads. You export a file. You drag it into the chat. You wait. Cowork eliminates that entire loop."
更重要的是 Cowork 能讀取你的舊報告來匹配格式、從上個月的試算表提取資料來建立這個月的、在任務中途參考你的品牌指南——全部因為你指向了正確的資料夾。
"Cowork can read your old reports to match your formatting. It can pull data from last month's spreadsheet to build this month's. It can reference your brand guidelines mid-task."
Context Files 策略（文章最重要的實務建議）：建立一個名為「Claude Context」的專用資料夾，裡面放三個檔案：
1. about-me.md — 你是誰、做什麼、什麼是工作中的成功
2. brand-voice.md — 你的溝通方式、常用短語、不喜歡的表達
3. working-style.md — 希望 Claude 如何運作：先問問題？長短輸出？檔案格式？
"Create a dedicated folder called 'Claude Context.' Inside it, build three files: about-me.md, brand-voice.md, working-style.md."
這些檔案會隨時間複利累積。每週你改進它們，Claude 就更擅長你的特定工作。這是整篇指南中最被低估的設定步驟。
"These files compound over time. Every week you refine them, Claude gets better at your specific work. It's the most underrated setup step."
🛠️ 實務應用
設定步驟：
1. 到 claude.com/download 下載桌面應用
2. 需要付費方案：Pro $20/月，Max $100+/月
3. 開啟應用，切換到頂部的 Cowork 標籤
4. 點擊資料夾圖示，選擇本地資料夾
5. 該資料夾所有檔案在 session 期間可被讀取
💻 第一個 Prompt
Read all the files in this folder completely. 
Then give me a summary of what you know 
about me, how I work, and what context 
you have access to.
⚠️ 思維轉換
核心洞察：
「停止思考如何寫更好的 Prompt，開始思考如何提供更好的 Files。」

這與 Context Engineering 的趨勢完全一致。你給 Claude 的 context 檔案品質越高，需要的 prompting 就越少。輸出品質從「通用 AI」提升到「聽起來像我會寫的東西」。
Part III: AskUserQuestion - 釐清而非猜測
Part III - Feature #2

讓 AI 問你，而不是替你猜

以下是每個其他 AI 在你給出不清楚任務時會做的事：它猜。而且很有自信地猜。它選一個詮釋、全力執行，然後給你一份精美的輸出——回答的卻是錯誤的問題。
"Here's what every other AI does when you give it an unclear task: it guesses. Confidently. It picks an interpretation, runs with it, and gives you a polished output that answers the wrong question."
Cowork 的做法不同。當它需要更多資訊時，它停下來產生一個表單：多選題、具體選項、結構化的提示，幫助你想清楚你真正要什麼。
"When it needs more information, it stops and generates a form. Multiple-choice questions. Specific options. A structured prompt that helps you think through what you actually want."
Nav Toor 現在用這個作為每個非瑣碎任務的起點。他不再試圖寫完美的 Prompt。他讓 Cowork 弄清楚它需要知道什麼。如果第一輪問題不夠，他說出來，Cowork 產生新的一組問題繼續迭代。
"I don't try to write perfect prompts anymore. I let Cowork figure out what it needs to know."
最重要的思維轉換：ChatGPT 訓練你寫更好的 Prompt。Cowork 訓練你給更好的 Context。一個是會貶值的技能，另一個會複利增長。
"ChatGPT trained you to write better prompts. Cowork trains you to give better context. One is a skill that deprecates. The other compounds."
💻 推薦 Prompt 模板
通用任務起手式： 或在任何 Prompt 結尾加上： 
DO NOT start working yet. First, ask me 
clarifying questions so we can define the 
approach together. Only begin once 
we've aligned.
💡 比喻說明
想像你請一位新員工幫你寫客戶提案。差的員工直接寫一份「看起來不錯」但完全不符合客戶需求的提案。好的員工先問你：「這個客戶的預算範圍？他們最在意什麼？過去合作過嗎？要正式還是輕鬆的語氣？」

AskUserQuestion 就是讓 Claude 成為那個好員工。
📚 核心概念
Context Window 的信心效應
Nav Toor 提到 Cowork 有超過 100 萬 token 的 context window。當 Claude 讀完所有相關檔案並透過問題釐清任務後，輸出一致地基於實際事實。這呼應了一個重要原則：Hallucination（幻覺）的根源通常是 context 不足，而非模型能力不足。 
I want to [YOUR TASK] so that 
[WHAT GOOD LOOKS LIKE].

First, read all uploaded files completely 
before responding.

DO NOT start executing yet. Ask me 
clarifying questions (use AskUserQuestion) 
to refine the approach. Only begin work 
once we've aligned.
Part IV: Plugins - 專業化外掛系統
Part IV - Feature #3

從通才到領域專家

沒有 Plugin 的 Cowork 是一個聰明的通才。但它不知道你的產業術語、團隊工作流程或你的角色需要的特定輸出。Plugins 是技能包、斜線命令和子代理的組合，專為特定工作職能設計。
"Without a plugin, Claude Cowork is a brilliant generalist. Plugins are bundles of skills, slash commands, and sub-agents designed for specific job functions."
Anthropic 在 2026 年 1 月推出了 11 個官方 Plugin：Productivity、Marketing、Sales、Finance、Data Analysis、Legal、Product Management、Customer Support、Enterprise Search、Biology Research 等。
影響立竿見影。Legal plugin 推出後，Thomson Reuters 單日下跌 16%——創下歷史最大單日跌幅。LegalZoom 跌了 20%。這就是 AI 真正開始做專業工作時的市場反應。
"When the Legal plugin launched, Thomson Reuters dropped 16% in a single trading session — its worst day on record. LegalZoom fell 20%."
安裝 Sales plugin，Claude 可以研究客戶、準備通話、起草外展序列。安裝 Data plugin，它能探索資料集、寫 SQL、建 dashboard、標記異常。不需要技術背景。
"Install the Sales plugin and Claude can research accounts, prep you for calls. Install the Data plugin and it explores your dataset, writes SQL, builds dashboards."
🛠️ 安裝步驟
1. 開啟 Cowork，點擊聊天欄的「+」按鈕
2. 點擊「Plugins」瀏覽資料庫
3. 或到 claude.com/plugins 查看所有可用外掛
4. 選擇適合你角色的 Plugin，點擊 Install
5. 在任何聊天中輸入「/」查看斜線命令
💻 範例 Prompt
Productivity Plugin: 
/productivity:start Let's review what I 
need to get done today and set up 
my task list.
Marketing Plugin: 
/marketing:draft-content Write a LinkedIn 
post about [topic]. Match the voice 
from my brand-voice.md file.
Data Plugin: 
/data:explore I've uploaded a CSV in 
this folder. Give me a summary, flag 
anomalies, suggest three analyses.
⚖️ 工具比較
Plugin vs Generic Prompt 的差異：
Plugin 啟動後的輸出明顯更結構化、更有主見。它知道你的職能需要什麼樣的好輸出。

這類似於 Claude Projects 的 Project Knowledge：給模型領域特定的知識和指令，輸出品質就會跳躍式提升。
Part V: Instructions & Connectors - 持久記憶與即時整合
Part V - Features #4 & #5

跨 Session 的記憶與工具串接

Instructions（指令）：Cowork 沒有跨 session 記憶。每次你開新對話，Claude 完全重新開始。Instructions 是你寫一次的常駐指令，在每個 Cowork session 開始時自動載入。Global Instructions 全域適用，Folder Instructions 在你選擇特定資料夾時適用。
"Instructions are standing orders you write once. They load automatically at the start of every Cowork session. Global instructions apply everywhere. Folder instructions apply when you select a specific local folder."
設定好 Instructions 後，Claude 在每個 session 開始時就已經知道你的名字、角色、溝通偏好、輸出預設和工作風格。你再也不需要重複解釋自己。
"Claude starts every session already knowing your name, your role, your communication preferences, your output defaults, and your working style."
Connectors（連接器）：典型 AI 工作流程涉及大量複製貼上。你截圖 Slack 對話、複製文件、貼到聊天中。Connectors 消除了所有這些。連接一次，Claude 可以在對話中引用來自 Slack、Drive、Notion 等 50+ 工具的即時資料。
"Link your tools once, and Claude can reference live data from them mid-conversation. No copy-pasting. No screenshots."
這在所有方案中免費。根據 Nav Toor 的經驗，Connectors 是 Cowork 中最被低估的功能。
"This is free on all plans. It's the most underused feature in Cowork, in my experience."
🛠️ Instructions 設定
Global Instructions 建議內容：
- 你的名字、角色和工作內容
- 溝通偏好（簡潔直接？不要填充？）
- 輸出預設（預設 .docx？.md？）
- 工作風格（先問問題再開始？）
- 避免事項（不用列點？不要太正式？）

路徑：Settings > Cowork > Edit Global Instructions
💻 Connectors 範例 Prompt
連接 Slack 後： 
Search my Slack messages from the last 
7 days and give me a summary of anything 
I need to follow up on. 
Organize by urgency.
連接 Google Drive 後： 
Find the most recent document about 
[project name] in my Drive. Read it 
and tell me the three most important 
things I need to know.
💡 比喻說明
Instructions = 員工手冊
想像你雇了一位新助理。每天早上他上班時都會自動閱讀你留的「員工手冊」，知道你的偏好和規則。你不需要每天重新培訓他。

Connectors = 辦公室門禁卡
這位助理有了門禁卡，可以進入 Slack 聊天室、Google Drive 檔案庫、Notion 資料庫。不用你幫他複製資料。
Part VI: Cowork 的限制（誠實版）
Part VI - Honest Assessment

六個你需要知道的限制

限制 說明 因應策略
無跨 Session 記憶 每次新對話完全從零開始，是最大摩擦點 用 Context Files + Global Instructions 彌補
關閉 App 任務中斷 Cowork 在桌面 App 中運行，關閉即停止 用睡眠模式代替關閉視窗；session 可存活睡眠
用量消耗快 複雜任務比普通聊天消耗更多配額 監控 Settings > Usage；重度使用者考慮 Max 方案
僅限桌面 無手機版、無瀏覽器版、無跨裝置同步 Context files 放在 iCloud/Dropbox 等雲端同步資料夾
不能生成圖片 文件、試算表、簡報可以，圖片不行 圖片用 Gemini Imagen，文件用 Cowork
Research Preview Agent 安全性仍在開發中 開啟刪除保護；敏感檔案讓 Claude 先確認再執行
Key Takeaways
Context > Prompts
最核心的思維轉換：停止優化 Prompt，開始投資 Context Files。三個 .md 檔案（about-me、brand-voice、working-style）就是你的 AI 協作基礎。這些檔案會隨時間複利累積。
五大功能的優先順序
File System Access 是基礎中的基礎（消除上傳/下載循環），AskUserQuestion 改變互動模式（從猜測到釐清），Plugins 實現專業化，Instructions 建立持久記憶，Connectors 串接即時資料。
30 分鐘設定投資
文章提供了明確的 30 分鐘設定流程：安裝（5min）→ Context Files（5min）→ Global Instructions（5min）→ 第一個任務（5min）→ Plugin（5min）→ Connector（5min）。一次性投資，每個 session 回報。
誠實的限制認知
無跨 session 記憶是最大痛點（用 Context Files 彌補）。桌面限定、用量消耗快、仍是 Research Preview。知道限制才能在對的地方用對的工具。
Practice Questions
Q1 概念理解 Nav Toor 說「ChatGPT trained you to write better prompts, Cowork trains you to build better context」。這兩種思維模式在實際工作中的差異是什麼？為什麼 Context 會「compound」而 Prompts 會「depreciate」？

Prompt 思維：每次任務都從零開始設計指令。你花 10 分鐘寫一個精巧的 Prompt，用完就丟。下次類似任務，你又重新寫。技能本身也會隨著模型進化而貶值——今天需要精心設計的 Chain-of-Thought，明天模型可能自動就做到了。

Context 思維：你一次投資建立 Context Files（個人資料、品牌聲音、工作風格），每個 session 自動載入。每週迭代改進這些檔案，Claude 的輸出品質就持續提升。這些資產不會因為模型升級而失效——相反，更好的模型能更好地利用你的 Context。

用製藥類比：Prompt 像是每批產品重新調配的手動參數；Context Files 像是經過驗證的 Master Batch Record，持續使用、持續改善、跨批次有效。

Q2 產業應用 如果你要在 CDMO 環境中導入 Cowork，File System Access 和 AskUserQuestion 分別對應哪些 GMP 合規需求？哪些場景適合「copilot mode」vs 哪些需要完全人工審查？

File System Access 在 CDMO 中的對應：Cowork 直接讀寫本地檔案的能力，在 GMP 環境下需要謹慎處理。21 CFR Part 11 要求所有電子紀錄有 audit trail。Cowork 目前沒有內建 audit trail，所以它適合非 GMP 文件（草稿、分析、培訓材料）但不適合直接操作受控文件（Batch Records、SOPs）。

AskUserQuestion 的合規價值：在受管制環境中，「AI 猜測後給出自信但錯誤的答案」是巨大風險。AskUserQuestion 的「先問再做」模式天然符合 ALCOA+ 的 Attributable 原則——每個決策都有明確的人類輸入。這是 copilot mode 的具體實現。

適用場景分級：Campaign Report 草稿、訓練材料 → Cowork 可直接產出（copilot）。CoA、Batch Record 摘要 → Cowork 草稿 + 人工逐行審查。偏差調查、CAPA → 人工主導，Cowork 僅做資料搜集。

Q3 批判思考 Nav Toor 將 Cowork 定位為「知識工作者的 Claude Code」。但他的文章缺少了什麼重要視角？你會如何用你的「2-3x discount principle」和「Removal Test」評估這篇文章的宣稱？

文章缺少的視角：

1. 成本效益量化：Nav Toor 說「60% 的知識工作」由 Cowork 完成，但沒有具體數據。每月 $20-200 的成本對應多少小時的節省？ROI 是多少？

2. 安全性深度：文章輕描淡寫「Research Preview」，但 Cowork 能讀寫你的本地檔案系統，這在企業環境中是重大資安考量。

3. 競品比較：完全沒提到 ChatGPT Desktop、Gemini、Copilot 等競品的對應功能。

2-3x Discount 應用：Nav Toor 說 Cowork「改變了他的工作方式」——打 2-3 折後，合理預期是「顯著改善了特定任務的效率」。Thomson Reuters 下跌 16% 的數據需要驗證——市場波動可能有其他因素。

Removal Test：如果明天 Cowork 消失，Context Files 策略仍然有效（可以用在 Claude Projects、ChatGPT Custom Instructions 等）。AskUserQuestion 的「先問再做」思維也是可移植的。Plugins 則高度綁定 Anthropic 生態系。

🎯 我的行動結論
✔ 核心洞察
Cowork 的核心價值不在任何單一功能，而在「Context Files + Instructions + AskUserQuestion」這三者組合形成的可累積系統。這與我一直強調的 Context Engineering 趨勢完全一致。我目前在 Claude Projects 中已經建立了類似的 Project Knowledge 系統——Cowork 的 File System Access 是這個系統的本地端延伸。最值得注意的思維轉換：從「寫更好的 Prompt」到「建立更好的 Context」，這不是 Cowork 獨有的洞察，而是整個 AI 使用方式進化的方向。
📈 立即應用
1. 這週建立三個 Context Files（about-me.md、brand-voice.md、working-style.md），即使暫時不用 Cowork，這些檔案也可以直接作為 Claude Projects 的 Project Knowledge 補充。2. 測試 AskUserQuestion 模式：在下一個複雜任務中，先加上「DO NOT start yet, ask clarifying questions first」，觀察輸出品質的差異。3. 評估 Cowork 在 CDMO 非 GMP 文件（培訓材料、Campaign Report 草稿）的適用性。
⚠ 需要補充
文章是典型的「功能介紹 + 設定教學」，缺少對以下面向的深入分析：(1) 企業資安——Cowork 的 File System Access 在受管制環境中的風險評估；(2) 與 Claude Code 的功能重疊和差異化定位；(3) 競品比較（特別是 ChatGPT Desktop 和 Microsoft Copilot）。此外，Plugins 的生態系統仍在早期，實際功能深度需要親自測試才能判斷。建議三個月後 Plugins 生態更成熟時重新評估。
