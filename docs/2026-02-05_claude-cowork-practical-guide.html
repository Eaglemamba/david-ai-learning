Executive Summary

這篇文章記錄了 AI 內容創作者 Ruben Hassid 從 ChatGPT 轉換到 Claude Cowork 的完整使用體驗。核心論點是 Claude Cowork 解決了四個長期困擾 AI 使用者的痛點：指令遵循不一致、長對話中的「無限迴圈」問題、記憶系統的過度擬合（overfitting），以及輸出僅限於文字而非實際檔案。文章透過五個真實案例展示 Claude Cowork 如何直接產生簡報、Excel、研究報告等可用檔案，對於希望將 AI 從「聊天機器人」升級為「實際工作夥伴」的專業人士具有參考價值。

Part I 指令遵循：從「忽略一半」到「完整執行」

作者描述了 ChatGPT 長期存在的核心問題：無論你提供多詳細的 prompt — 500 字的背景說明、具體需求、格式規則、想要和不想要的東西 — 它會忽略其中一半。

"You give it a detailed prompt. 500 words of context. Specific requirements. Formatting rules. Things you want. Things you don't want. It ignores half of it."

作者的解決方案是上傳超過 2,000 字的個人寫作風格檔案，描述他如何寫作、討厭什麼、從不使用的詞彙、偏好的結構。Claude Cowork 會完整讀取並遵循這些指令。

"I upload my voice profile. Over 2,000 words describing how I write, what I hate, phrases I never use, structures I prefer."

關鍵洞察：輸出品質取決於你能提供多少 context。如果 AI 忽略你的 context，你就永遠只能得到通用結果。

"The quality of your output depends on how much context you can give. If the AI ignores your context, you're stuck with generic results. Forever."

比喻說明
想像你僱用了一個助理，給他一份詳細的工作說明書。ChatGPT 就像一個助理會瀏覽說明書後說「我懂了」，但實際執行時只記得其中一半。Claude Cowork 則像一個會仔細研讀每一條、並在工作中持續參照的助理。
實務應用
Context 類型建議：
• Voice Profile（寫作風格檔）
• Copywriting Rules（文案規則）
• Reference Documents（參考文件）
• Domain-specific Guidelines（領域專業指南）

對於 CDMO 情境，可以準備：SOP 格式規範、技術寫作風格指南、常用術語對照表。

Part II 長對話穩定性：打破「無限迴圈」困境

作者描述了 ChatGPT 在複雜任務中常見的崩潰模式：進行 45 分鐘後開始重複自己、自相矛盾、或陷入不斷道歉卻產出相同錯誤輸出的迴圈。

"You're 45 minutes into a complex task. ChatGPT starts repeating itself. Or contradicting what it said earlier. Or getting stuck in a loop where it keeps apologizing and producing the same wrong output."

相比之下，Claude Cowork 可以進行 2 小時以上的複雜工作階段，包含來回修改、方向轉換、中途調整，而不會失去主題脈絡。

"I've had 2-hour sessions in Claude Cowork. Complex back-and-forth. Revisions. Pivots. New directions mid-conversation. It doesn't lose the thread."

重點提示：AI 自我提示技巧
當對話走偏時，作者使用這個 prompt 讓 Claude 來引導重新開始： 關鍵洞察：讓 AI 來提問你，而不是你不斷提示 AI。
核心概念：AskUserQuestion
Claude Cowork 可以生成結構化的問題表單來收集使用者輸入，這種「AI 提示人類」的模式翻轉了傳統的互動關係，特別適合需要多次迭代的複雜任務。
We are getting sidetricked. We need to start over without losing the context of this chat. First, generate a first AskUserQuestion form for me to answer so we get to decide what went wrong and where we must go instead. From my answer, generate a second AskUserQuestion form to now be able to complete the task.

Part III 無記憶設計：避免 Overfitting 的優勢

作者提出一個反直覺的觀點：沒有記憶反而是特性而非缺陷。ChatGPT 的記憶功能會創造「過度擬合」問題 — 它太過努力匹配你上週說的話，基於不再適用的舊 context 做出假設。

"ChatGPT remembers your past conversations. Sounds useful. But it creates what I call overfitting. It tries too hard to match what you said last week."

Claude Cowork 每次都從頭開始，沒有包袱、沒有三週前的假設。如果你想讓 Claude 知道什麼，就在這次對話中明確提供那個 context。

"If I want Claude to know something, I give it that context explicitly. In this conversation. For this task. With my favorite markdown files to support it."

比喻說明
想像你有兩種顧問：A 顧問會記住你過去所有對話，但經常把三年前的情況套用到今天；B 顧問每次見面都是全新開始，但你可以精確選擇要告訴他什麼。在快速變化的環境中，B 顧問的「精準 context 控制」往往更有價值。
實務應用：Context 管理策略
建立 Markdown 檔案庫：
• 個人介紹檔（角色、專長、工作背景）
• 專案情境檔（當前專案的關鍵資訊）
• 風格指南檔（偏好的格式和用語）

每次對話開始時，根據任務需求選擇性上傳，你控制完整 context，而非演算法猜測。

Part IV 實際檔案生成：從「複製貼上」到「直接產出」

這是作者認為最重要的差異：ChatGPT 給你文字，你複製貼上到其他工具再自己格式化。每一次都是這樣。

"ChatGPT gives you text. You copy that text. You paste it into another tool. You format it yourself. Every. Single. Time."

Claude Cowork 則直接建立實際檔案：要求簡報得到真正的投影片、要求試算表得到有公式的 Excel 檔、要求文件得到格式化好的 Word 檔。檔案直接出現在資料夾中。

"Claude Cowork creates the actual files. I ask for a presentation. I get slides. Real slides. I ask for a spreadsheet. I get an Excel file with working formulas."

重點提示
檔案生成的意義：
這不只是節省「複製貼上」的時間，而是改變了工作流程的本質。當 AI 可以直接產出可用檔案，它從「內容生成器」變成了「工作完成器」。

支援的檔案類型：
• 簡報（透過 Gamma MCP connector）
• Excel（含公式）
• Word 文件
• PDF
• Markdown

Part V 五個實戰案例：從理論到實踐

案例 1：從 Prompt 到簡報
作者結合 Claude Cowork 和 Gamma（AI 簡報工具）的整合，先用 AskUserQuestion 工具讓 AI 提問來確定大綱，內容確定後直接呼叫 Gamma MCP connector 生成簡報。

案例 2：大量文件閱讀篩選
面對出版社寄來的 4,992 個檔案，作者讓 Claude Cowork 分析並告訴他真正需要閱讀的內容及原因，而非自己一一過濾。

案例 3：Excel 公式自動化
Claude 不只可以生成 Excel 內容，還能建立包含有效公式的升級版試算表，直接輸出為 .xlsx 檔案。

案例 4：收據自動報銷
47 張出差收據，傳統方式需要 2 小時以上。作者將收據放入資料夾後下指令，10 分鐘內得到分類完成、計算好總額的 Excel 報表。模糊的 3 張收據被標記出來供手動確認。

"47 receipts from a work trip. The old way: 2 hours minimum... Time: 10 minutes total."

案例 5：研究簡報自動生成
撰寫 AI Agents 主題的電子報時，作者讓 Claude 搜尋、閱讀、摘要、組織，2 分鐘內得到按主題組織、含來源連結的 Markdown 檔案。

"I didn't open a single browser tab. Claude searched, read, synthesized, and organized. Time: 2 minutes."

案例共同模式
所有案例都展示了相同模式：輸入原始材料 → AI 處理 → 輸出可用檔案。這消除了傳統的「AI 產出 → 人工轉換 → 最終檔案」中間步驟。

Key Takeaways

指令遵循是基礎

再多的 prompt engineering 技巧都無法彌補工具本身忽略指令的問題。選擇能完整執行你 context 的工具。

無記憶 = 精準控制

與其依賴不可預測的記憶系統，不如建立自己的 context 檔案庫，每次對話精確選擇要提供的資訊。

檔案生成改變工作流程

當 AI 可以直接產出簡報、Excel、文件，它從「內容助手」升級為「工作完成器」。

工具是哲學，不只是功能

Claude Cowork 代表的是一種工作方式，學會這種模式後可以應用到任何未來的 AI 工具。

Practice Questions

Q1 概念理解 作者為什麼認為「無記憶」是優勢而非劣勢？這與傳統的 AI 產品設計思維有何不同？

核心論點：記憶系統會產生「過度擬合」（overfitting）— 系統試圖匹配過去對話的模式，可能基於不再相關的 context 做出錯誤假設。

思維轉換：傳統設計認為「記住使用者」等於「更好的體驗」，但在 AI 工作助手情境中，使用者需要的是「精準的 context 控制」而非「自動推斷」。每個任務可能需要完全不同的背景資訊，強制記憶反而會引入雜訊。

實務意涵：這意味著你需要建立自己的 context 管理系統（如 markdown 檔案庫），而非依賴平台的記憶功能。這是額外工作，但換來的是可預測性和精準度。

Q2 產業應用 在 CDMO 的 GMP 環境中，Claude Cowork 的「檔案直接生成」功能可能帶來什麼價值？有什麼合規性考量？

潛在價值：

• 偏差調查報告的初稿生成（直接輸出 Word 檔）

• 可行性評估的數據表格自動化（Excel 含公式）

• 技術轉移文件的格式化整理

• 訓練材料的簡報製作

合規性考量：

• 21 CFR Part 11：AI 生成的檔案仍需符合電子記錄的審計追蹤要求

• ALCOA+ 原則：AI 輸出需要人工審核以確保 Attributable 和 Accurate

• 變更控制：任何 AI 工具引入都需要通過變更管理流程

建議模式：AI 產生初稿 → 人工審核 → 正式文件系統登錄。AI 是加速器，不是決策者。

Q3 批判思考 作者聲稱 30 天沒開 ChatGPT，完全轉換到 Claude Cowork。這種「完全轉換」的論述有什麼潛在盲點？

潛在盲點：

• 使用情境侷限：作者是內容創作者，工作以文字、簡報、研究為主。對於其他情境（如程式開發、數據分析、創意設計），最佳工具可能不同。

• 功能比較不完整：文章沒有提到 GPT-4 的程式碼執行、DALL-E 整合、Voice Mode 等功能，也沒有比較 Gemini 的多模態能力。

• 蜜月期效應：任何新工具在初期都會感覺特別好用，需要更長時間才能評估真正的生產力影響。

• 成本效益未提：Claude Pro $17/月 vs ChatGPT Plus $20/月 的功能差異未完整分析。

更務實的立場：根據任務類型選擇最適合的工具，而非完全依賴單一平台。「工具是哲學」這句話的真正意涵是學會思維模式，而非綁定特定產品。

我的行動結論
核心洞察
這篇文章最有價值的觀點不是「Claude 比 ChatGPT 好」，而是「Context 控制權」的概念。在我的 CDMO 工作中，每個專案、每個客戶、每個 deviation 都有不同的背景脈絡。與其期待 AI 記住一切，不如建立模組化的 context 檔案庫，讓我精確控制每次對話的資訊輸入。這與我「copilot mode vs agent mode」的框架一致：在合規環境中，人類控制 context 等於控制輸出品質。
立即應用
本週行動：建立我的 CDMO Context 檔案庫，包含 (1) 個人專業背景檔 (2) SA25 Vanrx 技術規格摘要 (3) 常見偏差類型與調查框架 (4) 可行性評估標準模板。這些檔案可以在任何 AI 平台使用，不綁定特定工具。
需要補充
文章缺少對 Claude Cowork 檔案生成品質的深入評估 — 生成的 Excel 公式是否正確？Word 格式是否符合企業標準？這些需要我自己測試。另外，文章來自內容創作者視角，需要評估在技術文件、合規報告等專業情境的實際表現。建議 2 週後重讀，屆時應該已有實際使用經驗可對照。
