原文：核心痛點

Keeping up with AI news is overwhelming work and so tiring. You don't know who's going to lead this week and no idea what new tools will rise to the surface.

Every AI model tells they are better. Until another AI model takes over. The King throne only lasts for a week or two. That's how brutal AI news is.

To keep up with AI, I run an RSS reader with AI automation 24/7. It delivers every newsletter and blog post from 30+ sources I curated: the system works exactly as designed.

But it only covers what I already know.

"The real gap is the sources I don't know to follow yet. If I miss a few weeks, that blind spot turns into 3–5 hours of catch‑up just to get back to speed."

RSS only monitors sources I've manually added. If a breakthrough gets published on a blog I've never heard of, or discussed in a paper from a researcher I don't follow, or mentioned in a newsletter I'm not subscribed to—I miss it completely.

解析與評論

關鍵術語 Key Terms

RSS (Really Simple Syndication, 簡易資訊聚合)：一種標準化的內容訂閱格式，讓用戶可以訂閱特定網站的更新。就像訂報紙一樣，但只有你選擇訂閱的報紙才會送到你家。

Blind Spot (盲點)：你不知道自己不知道的資訊來源。這是資訊焦慮的根源——你無法訂閱一個你根本不知道存在的部落格。

比喻說明 Analogy

想像你經營一家餐廳，你有一份固定的供應商名單（RSS 訂閱）。這些供應商每天準時送貨，品質穩定。

但問題是：如果有一家新的農場種出了更好的有機蔬菜，而你根本不知道這家農場存在，你就永遠無法品嚐到這些食材。

這篇文章要解決的問題：建立一個「食材偵察員」，主動出去探索整個市場，找到你從未聽過但可能對你有價值的供應商。

重點提示 Key Insight

作者點出了一個關鍵區別：

  • RSS = Known Network (已知網絡)：被動監控你已經知道的來源
  • AI Agent = Discovery Engine (發現引擎)：主動搜尋整個網路，找到你不知道存在的資訊

這不是二選一，而是兩者互補。你需要 RSS 來追蹤固定來源，也需要 AI Agent 來發現新來源。

原文：系統架構

By the time you finish implementing this system, you'll have a fully automated AI news agent that:

  • Searches intelligently across the entire internet - Not limited to RSS feeds or pre-selected sources. Perplexity AI actively searches based on your custom topics and queries.
  • Stores everything in organized Google Sheets - Every research finding gets logged with metadata (source, date, topic, relevance score).
  • Sends weekly intelligence summaries to your email - OpenAI analyzes your accumulated findings and generates one clean summary email.
  • Runs completely on autopilot - Webhook trigger in Make.com kicks off the entire workflow.
  • Costs almost nothing to operate - Runs free within Make.com's limits. Perplexity and OpenAI API costs typically under $3-5 monthly.

Prerequisites you'll need:

  • Make.com account (free tier works)
  • Perplexity API access (minimum $5 in token credits)
  • OpenAI API key (minimum $5 in token credits)
  • Google Sheets (free)
  • Gmail account

Time to implement: 30-45 minutes for initial setup.

關鍵術語 Key Terms

Webhook (網路掛鉤)：一種讓不同應用程式之間自動傳遞資訊的機制。當某個事件發生（例如收到 Email），就會自動觸發另一個動作。

Token (代幣)：AI API 計費的基本單位。大約每 4 個英文字母 = 1 個 token。中文每個字約 2-3 個 token。

原文：Scenario 1 資料收集

The goal of this scenario is to begin requesting information from Perplexity to collect the data. We start this workflow with a Mailhook module.

Step 1: Use Mailhook to start your research

This automation begins by triggering a Mailhook and sending the request to Perplexity as the next module. You need to write your request in the email body. For example: "Research New AI tools this week."

Step 2: Perplexity to start researching

I chose Perplexity because it's great at deep research, scouring the internet to find relevant information based on your queries.

Key configuration fields:

  • Model: Choose "Sonar Pro" first. For more complex research, use Sonar Reasoning or Sonar Deep Research.
  • Max tokens: Set to 10,000
  • Web Search Options: Tweak between Medium and High

Step 3: Text Parser to fix JSON

Sometimes Perplexity returns output with leading "json" that breaks Google Sheets import. This module removes it.

Step 4: JSON Parse module

Parse the corrected JSON for downstream processing.

Step 5: Send to Google Sheets

Use Google Sheets: Bulk Add Rows (Advanced) to add all results.

解析與評論

Mailhook 觸發器
這就像一個私人信箱地址。當你發 Email 到這個地址，Make.com 就知道要開始執行自動化流程。這個設計的好處是：你可以從手機、電腦、任何地方隨時觸發研究。
Perplexity 搜尋
Perplexity 的優勢在於它不只是關鍵字搜尋，而是會「理解」你的問題，然後去網路上找相關資訊並整合。這比傳統 Google 搜尋更接近「研究助理」的功能。
JSON 清理
這是一個常見的技術問題：AI 有時會在 JSON 前面加上 "```json" 這樣的標記。這個步驟用 Regex (正規表達式) 來清除這些多餘的字元。
關鍵術語 Key Terms

JSON (JavaScript Object Notation)：一種資料交換格式，用花括號 {} 和方括號 [] 來組織資料。它是現代 API 最常用的資料格式，因為它既人類可讀，又容易被程式處理。

Regex (Regular Expression, 正規表達式)：一種用來描述文字模式的語言。例如 ```json\s*|\s*``` 這個 pattern 可以找到並移除 JSON 前後的標記。

比喻說明 Analogy

想像 Perplexity 是一位超強的研究員。你給他一個任務（「調查本週的 AI 新工具」），他就會：

  1. 去圖書館（整個網路）
  2. 找到相關的書籍和論文
  3. 閱讀並理解內容
  4. 寫成一份結構化的報告（JSON 格式）

JSON 清理這步就像是：研究員交報告時，有時會用信封裝著（加上 ```json 標記）。我們需要拆開信封，只保留報告本身。

重點提示 Key Insight

為什麼選擇 Perplexity 而不是直接用 ChatGPT？

Perplexity 專門為「即時網路搜尋」設計。ChatGPT 雖然也能搜尋，但 Perplexity 在以下方面更強：

  • 更新更快（幾乎即時）
  • 來源引用更完整
  • 專為研究任務優化
  • API 成本較低

原文：System Prompt 核心邏輯

The system prompt is the agent's brain. It tells Perplexity: "You're not just finding AI news - you're filtering signal from noise for people who actually USE AI tools, not just read about them."

EVALUATION CRITERIA:

  • Practical implementation potential for knowledge workers
  • Integration possibilities with existing workflows
  • Actual availability and accessibility (not vaporware)
  • Relevance to AI-augmented thinking, productivity, or learning

CATEGORIZATION FRAMEWORK:

  • "Tool Mastery" = Specific AI tools/platforms
  • "Workflow Mastery" = AI integration into daily work
  • "Thinking Mastery" = AI as cognitive partner
  • "Industry News" = Company announcements, funding
  • "Research" = Academic papers, benchmarks

RELEVANCE SCORING GUIDE:

  • 10 = Game-changing tool/feature available RIGHT NOW
  • 9 = High-value feature launching within days/weeks
  • 8 = Significant improvement with clear use case
  • 7 = Interesting new capability with medium-term relevance
  • 6 = Notable development worth tracking
  • 5 = Industry news with indirect impact
  • 4-1 = Decreasing relevance

解析與評論

關鍵術語 Key Terms

System Prompt vs User Prompt

  • System Prompt (系統提示)：定義 AI 的「人格」和「工作方式」。這是永久性的指令，告訴 AI「你是誰」以及「你應該如何思考」。
  • User Prompt (用戶提示)：每次互動時給的具體任務。這是「做什麼」的指令。

Vaporware (畫餅軟體)：指那些被大肆宣傳但實際上還不存在或無法使用的產品。這個詞提醒我們要專注於「真正可用」的工具。

比喻說明 Analogy

System Prompt 和 User Prompt 的關係，就像是：

System Prompt = 員工手冊：定義公司文化、工作標準、評估準則。員工入職時就要學會，之後每次工作都遵循這些原則。

User Prompt = 每日工作指令：今天要完成什麼任務、要調查什麼主題。

這個設計的優點是：你可以改變「做什麼」（User Prompt），但不需要重新教 AI「怎麼做」（System Prompt）。

分數 意義 範例
10 現在就能用的革命性功能 Claude 推出 100 萬 token 上下文
8-9 即將推出的高價值功能 GPT-5 下月發布
6-7 中期相關的有趣發展 公司公布產品路線圖
4-5 值得知道但不緊急 AI 公司獲得新融資
1-3 相關性低或純投機 純學術論文、遠期規劃
重點提示 Key Insight

Relevance Score 的設計智慧

這個評分系統回答了一個核心問題：「這條消息對我的『現在』有多重要？」

注意分數的邏輯：越是「立即可用」分數越高，越是「未來可能」分數越低。這反映了一個實用主義原則：資訊的價值與你能多快行動成正比。

原文：Scenario 2 摘要生成

Step 1: Get the data from Google Sheets

Filter and get data using Google Sheets: Search Rows (Advanced). The query returns rows where:

  • The date falls within the last 7 days
  • The relevance score is greater than or equal to 7

SQL-like Query:

where C >= date 'YYYY-MM-DD' and C <= date 'YYYY-MM-DD' and E >= 7 order by E desc

Step 2: Aggregate all data into Array

Bundle all selected rows into an array for downstream processing.

Step 3: OpenAI to summarize

The prompt transforms raw data into actionable insights by identifying patterns, connections, and emerging trends.

Output Structure:

  1. Opening Hook (1-2 sentences)
  2. The Big Picture (2-3 paragraphs)
  3. Must-Know This Week (3-5 developments)
  4. Good-to-Know (3-5 developments)
  5. Pattern Spotlight (2-3 paragraphs)
  6. Funding & Industry Moves
  7. Tools Worth Trying
  8. Newsletter Topic Ideas

Step 4: Send to Gmail

Set the whole scenario to run once a week, every Monday morning at 7 AM.

解析與評論

關鍵術語 Key Terms

SQL-like Query (類 SQL 查詢)：Google Sheets API 支援類似資料庫查詢的語法。這裡用的 WHERE、AND、ORDER BY 都是標準 SQL 關鍵字。

Array Aggregation (陣列聚合)：將多筆資料合併成一個陣列，方便一次性傳給下游處理。這樣 OpenAI 可以「看到全貌」再做分析。

比喻說明 Analogy

Scenario 2 就像一位資深編輯在做週報：

  1. Step 1 (篩選)：從檔案櫃中拿出本週的文件，只挑選評分 7 分以上的
  2. Step 2 (整理)：把所有文件放在桌上，讓編輯可以一次看完
  3. Step 3 (寫作)：編輯閱讀所有資料，找出模式，寫成一篇有洞察的報告
  4. Step 4 (發送)：把報告寄給老闆
重點提示 Key Insight

為什麼 OpenAI 的 Prompt 如此複雜？

這個 prompt 的設計目標不是「列出本週新聞」，而是「像資深分析師一樣思考」。它要求 OpenAI：

  • Pattern Recognition：找出多家公司在解決類似問題
  • Relationship Mapping：分析競爭動態、生態系佈局
  • Signal vs Noise：區分真正創新 vs 純粹炒作

這就是 Prompt Engineering 的精髓：你不是在「使用 AI」，而是在「訓練一個專家」。

實務應用 Practical Application

報告結構設計的智慧：

  • Opening Hook：用最吸引人的發現開場，決定讀者會不會繼續讀
  • Must-Know vs Good-to-Know：幫忙讀者判斷優先順序
  • Pattern Spotlight：這是分析師的核心價值——看到別人看不到的連結
  • Newsletter Topic Ideas：如果你是內容創作者，這直接給你內容靈感

重點提示 Key Insight

可擴展性的設計思維

這個系統的聰明之處在於它分離了「收集」和「分析」：

核心概念總結 Key Takeaways

1. RSS vs AI Agent

RSS 監控已知來源，AI Agent 發現未知來源。兩者互補，不是替代關係。

2. System vs User Prompt

System Prompt 定義「如何思考」，User Prompt 定義「做什麼」。分離兩者讓系統更靈活。

3. 收集 vs 分析分離

Scenario 1 專注收集，Scenario 2 專注分析。這種架構更容易擴展和維護。

4. 複利效應

系統運行越久，資料庫越有價值。這不只是省時間，而是在建立情報資產。
