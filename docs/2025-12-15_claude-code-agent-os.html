Introduction: Beyond the Developer Tool

Most people think Claude Code is for engineers who want to build apps faster. I thought that too. But as much as I wanted to try, I already had a coding setup that worked. Cursor handled my vibe-coding projects just fine. Claude web/desktop was perfect for brainstorming, researching, and writing. My workflows were humming along smoothly.

So when Claude Code launched, my honest reaction was: "Why would I complicate a system that's already working?"

"Claude Code might be the best writing and research assistant I've ever used."

I was working on a complex article and drowning in research. Ten PDF files, screenshots from different experiments, random notes scattered everywhere. You know that feeling when you can't remember which file contains that perfect quote? Or when you have documents open and you're losing track of how they all connect?

On a whim, I pointed Claude Code at my messy research folder. Instead of me explaining what each file contained, Claude Code read everything and immediately started connecting dots I'd missed.

What makes Claude Code different:

  • Agentic: It understands context better and proactively completes tasks for you
  • Fast: Since it lives in your terminal, latency is low, so it thinks and executes faster
  • Direct file access: This is the big one—it can access and edit your files directly

關鍵術語解析

Claude Code：Anthropic 推出的終端機（Terminal）AI 工具，讓 Claude 能直接在你的電腦上讀取、編輯檔案，執行命令。與一般的 Claude Web/Desktop 不同，它能主動操作你的本機環境。

Agentic（代理式）：指 AI 具備自主完成任務的能力，不只是被動回答問題，而是能主動規劃、執行多步驟工作。

Terminal（終端機）：電腦的命令列介面，讓你用文字指令控制電腦，而非圖形化點擊。

比喻說明

想像你請了一位新助理。

傳統 Claude Web 像是「遠端視訊助理」——他很聰明，但只能透過螢幕看你描述的東西，無法直接翻閱你桌上的文件。你得把文件內容唸給他聽或截圖傳過去。

Claude Code 則像是「坐在你隔壁的助理」——他可以直接打開你的檔案櫃、翻閱資料、整理文件，甚至幫你把筆記歸檔。你不用解釋「那份報告在第三個抽屜」，他自己就能找到。

重點提示

作者原本的工作流已經運作良好（Cursor 寫程式 + Claude Web 做研究），但 Claude Code 帶來的關鍵突破是：直接檔案存取。

這解決了知識工作者最常見的痛點：

  • 研究資料散落各處，難以整合
  • 每次開新對話都要重新解釋專案背景
  • AI 無法直接看到你的工作脈絡

IDE 是什麼？

IDE（Integrated Development Environment，整合開發環境）：一種軟體工具，把程式碼編輯、檔案管理、終端機等功能整合在一個介面裡。

常見的 IDE 包括：

  • VSCode：微軟出品，免費，適合各種程式語言
  • Cursor：基於 VSCode，內建 AI 功能，適合 AI 輔助開發

即使你不寫程式，IDE 也能當作檔案管理器使用，讓你視覺化地瀏覽專案結構。

Getting Started with Claude Code

Step 1: Install Claude Code

Open your Terminal, copy and paste this code:

npm install -g @anthropic-ai/claude-code

Before installing Claude Code, make sure you have NodeJS 18+ installed.

Step 2: Set up your project structure

First create a folder on your computer. In my case, it's "AI Maker Newsletter."

Step 3: Use your IDE for visual file management

Open Cursor and click "Open Project" to access your project folder. To access and manage your project files while using Claude Code, you need to use an IDE like VSCode or Cursor.

Step 4: Test your setup and activate Claude Code

Go to the Terminal tab inside your IDE, simply type "claude," and Claude Code will be activated.

Once you have both Claude Code running and your project folder open in your IDE, try this simple test:

  • In your terminal with Claude Code activated, type: "Show me what files are in this project"
  • In your IDE, create a new text file called "test.md"
  • Ask Claude Code: "What did I just add to this project?"

Claude.md: Your Project's Memory System

Most people treat Claude Code like a chat session that happens to access files. They start fresh every time, explain their project context over and over, and wonder why Claude Code doesn't seem that much better than regular Claude.

The secret is claude.md—a special file that acts as your project's persistent memory system.

"claude.md is Claude Code's brain for your specific project. Every time you start a session, Claude Code reads this file first to understand what you're working on, how you work, and what your goals are."

What claude.md actually does:

  • Stores project context that persists across all sessions
  • Remembers your working style and preferences
  • Contains your specific instructions for how Claude should help you

How I set up claude.md for my newsletter project

First, I gathered all my newsletter-related files into my "AI Maker Newsletter" project folder:

  • Performance data from past newsletters
  • All my published posts
  • My about page and newsletter description
  • Welcome email
  • Collection of my social notes
  • Reader feedback and comments
  • Content strategy documents

Once everything was in the folder, I opened my terminal with Claude Code activated and ran this command:

/init

That's it. Claude Code read through my entire project folder and automatically generated a comprehensive claude.md file.

Persistent Memory（持久記憶）：不同於一般聊天會話結束就遺忘的情況，claude.md 讓 Claude Code 在每次新會話開始時都能「記住」你的專案脈絡。

  • SOP 流程 = 你希望 Claude 如何協助你的具體指示

有了這本手冊，新員工（Claude Code）第一天上班就能立刻進入狀況，而不是每天都要你從頭解釋一遍。

Slash Commands: Shortcuts to Your Work

Slash commands are one of Claude Code's powerful weapons. They're pre-built shortcuts that tell Claude Code exactly what type of help you need, without you having to explain the context every single time.

Think of slash commands like having a toolbox where each tool is designed for a specific job.

Top 5 built-in slash commands:

  • /agents - Shows you active sub-agents working on different parts of your project
  • /compact - Clear conversations history but keep a summary in the context
  • /exit - Cleanly closes your Claude Code session
  • /mcp - Lists and manages your Model Context Protocol connections
  • /memory - Shows what Claude Code remembers about your current project session

Creating custom commands

Here's mine that saves me hours every week:

My /quick-edit command:

Help me quickly improve this existing AI Maker newsletter content by:

1. Strengthening the personal hook/opening
2. Making the framework clearer and more actionable  
3. Improving transitions between sections
4. Enhancing the conclusion with stronger next steps
5. Ensuring voice consistency throughout

Please review the content and suggest specific edits that will increase engagement and value while maintaining The AI Maker brand voice and approach.

Other custom commands I use:

  • /generate-ideas - Analyzes audience engagement data and suggests new newsletter ideas
  • /seo-optimize - Reviews past posts and suggests SEO improvements
  • /perspective-analysis - Reviews content through five different perspectives

關鍵術語解析

Slash Commands（斜線指令）：以 / 開頭的快捷指令，類似於 Slack、Discord 等應用中的斜線指令概念。預先定義好的指令讓你不用每次都打一長串說明。

Custom Commands（自訂指令）：你可以建立自己的斜線指令，把常用的複雜 Prompt 封裝成一個簡單的指令。

比喻說明

斜線指令就像是「速食店的套餐」：

沒有套餐時，你要說：「我要一個漢堡、不要洋蔥、多加番茄醬、配薯條中份、可樂去冰......」

有套餐後，你只要說：「一號餐」，服務生就知道所有細節。

/quick-edit 就是作者的「一號餐」——包含了五個編輯要點、品牌聲音要求，一次指令搞定。

重點提示

為什麼自訂指令這麼重要？

根據原文，作者每次手動輸入編輯需求需要寫：

「Please help me improve this article by making the introduction stronger, clarifying the main points, improving transitions, and ensuring the conclusion has clear next steps while maintaining my writing voice.」

有了 /quick-edit 後，只需要三個字元就能完成同樣的指示。這種效率提升在日復一日的工作中會累積成巨大的時間節省。

設計好用自訂指令的原則

  1. 單一目的：每個指令專注解決一種問題
  2. 具體明確：列出具體步驟，而非模糊指示
  3. 包含品質標準：說明什麼是好的產出
  4. 整合品牌/風格：讓 AI 知道要符合你的聲音

練習思考

問題：假設你是社群媒體經理，每天要把長文轉成 5 篇短貼文。你會如何設計一個 /repurpose 指令？

思考要包含：

  • 哪些平台？（LinkedIn、Twitter、IG......）
  • 每篇的字數限制？
  • 語調風格？
  • 是否需要 CTA（Call to Action）？

Plan Mode: Think Before You Act

Those of you who have done vibe-coding before can relate to this problem: we ask for a brainstorming session and AI starts coding without us asking for it. This also happens with writing. Half the time, we realize halfway through that it's going in the wrong direction, but by then it has already modified files or created content we didn't want.

"Plan mode solves this completely. Instead of Claude Code jumping straight into action, it first shows you a detailed plan of what it's going to do, then waits for your approval before executing anything."

How plan mode can save you from accidental AI execution

When you're working on research or writing projects, you want to see the thinking process before the execution.

Let's say I ask Claude Code to "generate 5 LinkedIn posts." Without plan mode, Claude Code might start editing files around immediately. With plan mode, it first shows me its plan—and I can see exactly what Claude Code is planning to do.

How to activate plan mode

Plan mode is not running by default. To activate it, simply select Shift + Tab on your keyboard until you see the plan mode is on.

When to use plan mode

For my newsletter work, I always want plan mode when I'm asking Claude Code to:

  • Analyze multiple research sources and create summary documents
  • Reorganize content folders or research materials
  • Process reader feedback and suggest content improvements
學習筆記 Commentary

關鍵術語解析

Plan Mode（計畫模式）：Claude Code 的一種執行模式，在這個模式下，AI 會先展示它打算做什麼，等你確認後才真正執行。這是一種「人類在迴圈中」（Human-in-the-Loop）的設計。

Vibe-coding：一種用自然語言描述需求、讓 AI 生成程式碼的工作方式。特點是快速但有時候會「失控」。

比喻說明

想像你請室內設計師改造房間：

沒有 Plan Mode：你說「我想要現代風格」，設計師直接開始拆牆、搬家具。等你回來發現：「等等，我的收藏牆被拆了！」

有 Plan Mode：設計師先給你看設計圖：「我打算拆這面牆、把沙發移到這邊、換掉窗簾......」你可以說：「不要拆那面牆，其他可以。」然後才開始動工。

重點提示

Plan Mode 解決的核心問題：

Agentic AI（代理式 AI）的特點是「主動執行」，這帶來效率但也帶來風險——AI 可能會：

  • 修改你不想改的檔案
  • 產生你不需要的內容
  • 走向你沒預期的方向

Plan Mode 在「效率」和「控制」之間找到平衡：你仍然能享受 AI 的主動性，但在關鍵步驟前有確認機會。

何時該用 Plan Mode？

使用 Plan Mode 可以跳過
涉及多個檔案的操作 單純的問答查詢
會修改現有內容 從零開始創建新內容
複雜的多步驟任務 簡單的格式調整
不可逆的操作（刪除、移動） 可輕易撤銷的操作

Sub-Agents: Your AI Team Working in the Background

One of the most mind-blowing features of Claude Code is sub-agents. These are smaller, specialized AI assistants that Claude Code can spawn to handle specific tasks while you work on something else.

What makes sub-agent so special?

  • Has a specific purpose and expertise area
  • Uses its own context window separate from the main conversation
  • Can use specific tools like MCP
  • Has its own system prompt that guides its behavior

Why sub-agents change how you work

Normally, the way we work with AI is linear: you ask for something, wait for the response, then ask for the next thing. Sub-agents break this pattern completely.

Instead of stopping your writing to research a topic, you can deploy a sub-agent to handle the research while you keep writing.

How I use sub-agents for newsletter automation

Every morning, I ask the sub-agent to:

  • Scan daily AI news sources using Perplexity MCP
  • Filter content specifically for my audience
  • Suggest personalized content ideas based on findings and past performance
  • Output everything to a specific file in my "Content Ideas" folder
"This sub-agent has saved me at least 5 hours per week of manual research and idea generation."

My sub-agents:

  • Substack Notes Generator: Crafts 10 Substack Notes tailored to my project
  • Tool Analysis Review: Runs competitive analyses of AI tools
  • Daily AI News Summary: Searches latest AI news and explains why each story matters
學習筆記 Commentary

關鍵術語解析

Sub-Agent（子代理）：Claude Code 能夠「召喚」的專門化小型 AI 助手，各自有獨立的任務範圍、工具權限、系統提示。

Context Window（上下文窗口）：AI 一次能「記住」的資訊量。子代理有自己獨立的上下文，不會佔用主對話的空間。

System Prompt（系統提示）：給 AI 的角色設定和行為指南，決定這個 AI 會「扮演」什麼角色。

比喻說明

想像你是一家小公司的老闆，只有你一個人時：

沒有 Sub-Agent：你寫文章寫到一半，想到需要查資料，就得停下來去查。查完資料，思路已經斷了，又要花時間重新進入狀態。

有 Sub-Agent：你有三個專職員工——研究員（Daily AI News Summary）、社群編輯（Substack Notes Generator）、市場分析師（Tool Analysis Review）。你繼續寫文章，他們各自在旁邊默默工作，做完就把結果放到你的桌上。

這就是「平行處理」vs「串行處理」的差別。

重點提示

Sub-Agent 的威力在於：打破線性工作流

傳統 AI 對話是「問→答→問→答」的序列模式。Sub-Agent 讓你能：

  • 同時進行多項任務
  • 後台自動執行重複性工作
  • 專注在高價值的創意工作，把例行任務外包給 AI

作者每週省下 5 小時，換算成一年就是 260 小時——超過 10 個整天！

設計有效 Sub-Agent 的原則

根據原文，好的 Sub-Agent 需要：

  1. 明確的重複性任務：有固定模式的工作最適合
  2. 清楚的輸出要求：結果要放在哪裡、用什麼格式
  3. 定義好的篩選標準：什麼來源、什麼條件要納入

Output Styles: Training Claude Code to Think Like You

Output styles might be Claude Code's most underrated feature. Most people never realize they can train Claude Code to respond in completely different ways depending on the type of work they're doing.

"I like to think output styles as different hats Claude Code can wear. When you're brainstorming, you want creative, exploratory responses. When you're analyzing data, you want structured, analytical output."

Why output styles make a huge difference

The problem with most AI conversations is that you get one-size-fits-all responses. Claude gives you the same type of answer whether you're asking for creative ideas or strategic analysis.

My three specialized output styles:

1. Growth Strategist Mode

  • Specific growth metrics and benchmarks
  • Actionable strategies with timeline estimates
  • Competitive analysis and positioning insights
  • Growth experiment suggestions with success criteria

2. Funnel Conversion Mode

  • Content gaps between free and paid tiers
  • Value proposition optimization for paid content
  • Conversion sequence improvements
  • Pricing psychology and positioning strategies

3. Social Media Strategist Mode

  • Content repurposing for maximum social impact
  • Platform-specific optimization strategies
  • Viral content patterns and timing
  • Cross-platform funnel optimization
學習筆記 Commentary

關鍵術語解析

Output Styles（輸出風格）：本質上是 Claude Code 的 System Prompt（系統提示），但概念更像是「角色切換」。不同的 Output Style 讓 Claude Code 以不同的思維框架來回應你。

System Prompt：給 AI 的「角色說明書」，定義它應該如何思考、回應、著重什麼。

比喻說明

Output Styles 就像「戴不同的帽子」：

  • Growth Strategist 帽子：戴上這頂帽子，Claude 變成專注成長指標的顧問，會給你數據、基準、實驗建議
  • Funnel Conversion 帽子：變成轉換率專家，專注在「如何讓免費用戶付費」
  • Social Media 帽子：變成社群經理，思考的是「這內容在哪個平台會火」

同樣的問題「如何增加訂閱者」，不同帽子會給出完全不同角度的建議。

重點提示

為什麼這是「最被低估的功能」？

因為大多數人從未想過要「切換 AI 的思維模式」。他們期待一個萬能的 AI，結果得到的是「什麼都會一點，但都不夠專精」的回答。

原文的例子很說明問題：

沒有 Output Style 有 Growth Strategist Style
「要成長，就要持續產出高品質內容、保持一致性......」（通用建議） 「根據你目前的指標，瓶頸在 X，建議執行 A 實驗，預計 2 週見效......」（具體策略）

如何設計自己的 Output Style？

  1. 識別你的「思維模式」：你工作時有幾種不同的「帽子」？創意、分析、策略、執行？
  2. 定義每種模式的特徵：
    • 優先考慮什麼資訊？
    • 回答要多詳細？
    • 偏好什麼格式（條列、段落、框架）？
    • 成功的標準是什麼？
  3. 測試和迭代：實際使用，觀察哪些回應不符預期，持續調整

MCP Integration: Extending Claude Code Beyond Your Files

Just like Claude web and desktop, you can use MCP (Model Context Protocol) to connect Claude Code with multiple external apps and services.

The difference with Claude Code is that MCP integrations work seamlessly within your project context. Instead of switching between Claude desktop and your project files, everything happens in one unified workspace.

My current MCP setup for research workflows:

Perplexity MCP

This lets Claude Code perform real-time web searches and analysis without me leaving my project. When I'm working on an article about a new AI tool, Claude Code can instantly research the tool, find user reviews, and gather competitive information—all while maintaining context about my specific article and audience.

Firecrawl MCP

This integration lets Claude Code extract and analyze content from websites directly. When I find a relevant article or tool documentation, Claude Code can pull the full content, analyze it for key insights, and integrate findings into my research notes automatically.

Setting up MCP with Claude Code

The setup process is similar to what you've seen with Claude desktop—you configure the MCP connections through Claude Code's settings, authenticate your services with API key, and start using them in a conversation.

The key difference is that Claude Code's MCP integrations understand your project context, so they can automatically organize results, maintain consistency with your existing files, and follow the patterns established in your claude.md file.

MCP（Model Context Protocol，模型上下文協議）：Anthropic 開發的標準協議，讓 AI 模型能夠連接和使用外部工具、服務。可以把它想成「AI 的 USB 接口」——任何遵循這個標準的工具，都能插上來讓 AI 使用。

比喻說明

把 Claude Code 想像成你的「大腦」，MCP 就是讓大腦能夠「伸出手」觸及外界的神經連結：

  • Perplexity MCP = 讓你的大腦能即時「看到」網路上最新資訊
  • Firecrawl MCP = 讓你的大腦能「深讀」任何網頁的完整內容
  • Notion MCP（作者以前用的）= 讓你的大腦能「存取」你的知識庫

有了 MCP，Claude Code 不再是一個「只能看本機檔案」的助手，而是能觸及整個數位生態系的智能中樞。

Claude Code MCP vs Claude Desktop MCP 的關鍵差異

GitHub + Obsidian: 24/7 AI Assistant Across All Devices

"This is the most insane feeling I've had recently with AI. Everything we just covered—slash commands, sub-agents, MCP integrations, output styles—can now be accessed from anywhere, anytime, through your mobile phone."

The mobile breakthrough: GitHub issues as Claude Code commands

By connecting your Claude Code project to GitHub, you can use GitHub's mobile app to command Claude Code through issue tickets.

Want Claude Code to repurpose your latest newsletter into LinkedIn posts while you're on the train? Create a GitHub issue, mention Claude, and describe what you need. Claude Code processes your request, executes the work using all your project context then updates the issue with results.

The workflow:

  1. Open GitHub mobile app
  2. Create new issue in your project repository
  3. Write your request description
  4. Write a comment: "Hey @claude, please handle this issue."
  5. Results delivered - Claude updates the issue with completed work

Obsidian: Your project data everywhere

Unlike Notion or other cloud-based tools, Obsidian stores everything as local markdown files. Since my entire newsletter project uses markdown files, I can sync the complete project folder through Obsidian and access it from any device.

Pro tip for voice capture:

I use Wisprflow through Apple Shortcuts to record voice memos that automatically get inserted into my Obsidian vault. When I have a content idea while walking or driving, I just trigger the shortcut, record my thoughts, and the transcribed memo appears in my brain-dump folder.

關鍵術語解析

GitHub：原本是程式碼版本控制平台，但這裡被創意運用為「遠端任務派發系統」。透過 GitHub Issues 功能，你可以從任何裝置指派任務給 Claude Code。

GitHub Issues：GitHub 的任務追蹤功能，原本用於回報 Bug 或功能需求。

Obsidian：一款本機優先的筆記軟體，所有資料以 Markdown 檔案形式儲存在你的電腦上，而非雲端伺服器。

Wisprflow：語音轉文字工具，可搭配 Apple Shortcuts 自動化工作流程。

比喻說明

這個工作流就像是建立了一個「24 小時待命的遠端秘書系統」：

GitHub = 你和秘書之間的「交辦事項信箱」，你隨時可以丟任務進去

Claude Code = 那位秘書，會讀取任務、執行工作、回報結果

Obsidian = 你和秘書共用的「檔案櫃」，不管誰在哪裡都能存取同一份資料

Wisprflow = 錄音筆，讓你的靈感能即時轉成文字進入系統

重點提示

這個架構的突破性意義：

傳統上，Claude Code 是「桌面工具」——你得坐在電腦前才能用。但透過 GitHub + Obsidian 的組合：

  • 輸入端（GitHub Issues）：任何裝置都能下達任務
  • 處理端（Claude Code）：在你的電腦/伺服器上自動執行
  • 輸出端（Obsidian Sync）：結果同步到所有裝置

你在通勤時想到點子 → 用手機發 GitHub Issue → 到辦公室時，文章初稿已經在 Obsidian 裡等你了。

總結：將 Claude Code 打造成個人 AI 作業系統

核心概念回顧

  • Claude Code 不只是工程師的工具，更是寫作者和研究者的強力助手
  • claude.md 是專案的「記憶核心」，讓 AI 能跨會話記住你的脈絡
  • Slash Commands 把複雜指令封裝成一鍵執行的快捷方式
  • Plan Mode 在效率和控制之間取得平衡，先看計畫再執行
  • Sub-Agents 打破線性工作流，讓多項任務平行進行
  • Output Styles 讓 AI 切換不同的「思維模式」來回應你
  • MCP 擴展 AI 能觸及的範圍，連結外部工具和服務
  • GitHub + Obsidian 實現隨時隨地的 24/7 工作流

分階段學習路徑

  • Phase 1：安裝 Claude Code，設定專案資料夾，執行 /init 生成 claude.md
  • Phase 2：建立你的第一個自訂 Slash Command，針對最常做的任務
  • Phase 3：在複雜任務中使用 Plan Mode，體驗「先看計畫」的好處
  • Phase 4：部署第一個 Sub-Agent，處理固定的背景任務（如每日新聞摘要）
  • Phase 5：加入 MCP 整合和行動工作流，打造完整的 AI 作業系統

實作檢查清單

  • 安裝 Node.js 18+ 和 npm
  • 安裝 Claude Code：npm install -g @anthropic-ai/claude-code
  • 建立專案資料夾，放入相關檔案（過去的作品、風格指南、受眾資料）
  • 執行 /init 生成 claude.md
  • 建立至少一個自訂 Slash Command
  • 學會切換 Plan Mode（Shift + Tab）
  • 設計並部署一個 Sub-Agent
  • 設定至少一個 MCP 整合（推薦從 Perplexity 開始）
  • （進階）設定 GitHub + Obsidian 行動工作流

最終思考

作者在文末提出了一個很重要的現實觀點：Claude Code 不會解決所有生產力問題，也會有讓你挫折的時刻。但一旦你體驗過「AI 真正理解你的專案脈絡」是什麼感覺，回去用傳統的 AI 對話就會感覺「像是單手被綁住工作」。

關鍵不在於一次學會所有功能，而是逐步建立——從最能立即解決你痛點的功能開始，慢慢擴展你的 AI 作業系統。
