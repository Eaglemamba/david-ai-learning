CLAUDE.md 檔案是最被低估的功能之一。在專案根目錄建立這個 markdown 檔案，Claude 每次對話都會自動讀取。你可以在裡面放專案規則、偏好設定、上下文資訊。

"Create a markdown file in your project root with project rules, preferences, and context. Claude reads this automatically every session."

Connectors 連接器讓你直接在對話中操作常用應用。Google Calendar 和 Notion 是最實用的兩個連接——可以直接查詢行事曆、搜尋筆記資料庫。

"Settings → Connectors - This is how you prompt your favorite apps. I personally use Google Calendar & Notion connectors daily."

Haiku 4.5 是目前最快的 Claude 模型，適合日常任務。如果你的任務不需要深度推理，用 Haiku 可以大幅提升回應速度，特別適合 Chrome 擴充功能的快速回覆場景。

"Haiku 4.5 - This is Claude's latest model and fastest LLM - imo, the best Claude model for daily tasks."

Plan Mode 是 Claude Code 的規劃模式。按 Shift + Tab 進入，用於在開始編碼前進行專案規劃。這是「先思考再執行」的最佳實踐。

"Shift + Tab to enter 'Plan' mode. This is the mode you want to be in for planning projects."

/doctor 指令是環境診斷工具，能檢測缺失的依賴項和配置問題。當專案出錯時，這應該是你第一個執行的指令。

"Doctor Command - /doctor diagnoses environment issues, missing dependencies, and configuration problems. Run this first when things break."

MCP (Model Context Protocol) 連接是 Claude Code 的擴展能力。如果 Claude 無法完成某項任務，MCP 連接通常能幫上忙。這是 Anthropic 的開放協議，讓 AI 工具能與外部服務互動。

"MCP - Learn the ins and outs of MCP connections. If Claude can't do something, an MCP connection will likely be able to help."

Cowork 是 Claude 的桌面代理功能，可以存取你的檔案系統和瀏覽器。但要注意：Cowork 消耗 credits 很快。建議先用 Sonnet 4.5 在標準聊天中規劃，再切換到 Cowork 執行。

"Usage Tip - Cowork burns through credits. Use standard chat with Sonnet 4.5 to plan, THEN switch to Cowork to execute."

隔離存取是安全的關鍵。建立專用的「Cowork」資料夾，只授權 Claude 存取該目錄，而非整個 Documents 資料夾。這限制了潛在風險。

"Isolate Access with Folders - Create a dedicated 'Cowork' folder and grant Claude access only to that directory."

Cowork 支援多執行緒並行。你可以同時啟動研究專案、文件格式化、數據分析等不同任務，它們會在不同 threads 中平行執行。

"Multiple Threads - Cowork can run multiple tasks simultaneously."

Practice Questions | 練習題

Q1 概念理解 文章提到的「Model Stacking」概念，與你在 CDMO 環境中使用 AI 工具有什麼關聯？

在 CDMO 環境中，Model Stacking 的概念可以應用於：

  • Perplexity + Claude：用 Perplexity 搜集法規更新資訊（如 FDA Guidance），再匯入 Claude Project 進行深度分析和合規性評估
  • ChatGPT 語音 + Claude：在實驗室或產線走動時用語音輸入偏差描述，再將逐字稿匯入 Claude 進行根因分析
  • Excel Gemini + Claude：用 Gemini 處理批次紀錄數據，再用 Claude 生成 ALCOA+ 合規的報告

核心原則是：選擇每個工具的強項，而非試圖讓單一工具做所有事。

Q2 產業應用 如果你要為你的團隊建立一個 CLAUDE.md 檔案，會包含哪些製藥產業特定的內容？

CDMO 專用的 CLAUDE.md 可能包含：

  • 法規框架：「所有文件輸出必須符合 21 CFR Part 11 和 ALCOA+ 原則」
  • 術語定義：常用縮寫如 OOT、OOS、CAPA、PQ、PPQ 的展開和上下文
  • 專案上下文：「我們使用 SA25 Vanrx isolator 進行 aseptic fill-finish」
  • 輸出格式：「偏差報告使用 [Root Cause → Contributing Factors → CAPA] 結構」
  • 敏感性聲明：「避免在回應中包含具體客戶名稱或批號」
Q3 批判思考 這篇文章有什麼明顯的缺失或限制？哪些建議可能不適用於受監管的產業環境？
