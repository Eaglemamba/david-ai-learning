深度解析 Commentary

核心概念解析 Key Concepts

Claude Skill（Claude 技能）

一種模組化、自我包含的套件，透過提供專業知識、工作流程和工具來擴展 Claude 的能力。把它想像成給 Claude 的「新員工訓練手冊」——讓通用型 AI 變成特定領域的專家。

SKILL.md（技能主檔）

每個 Skill 的核心指令檔案，包含 YAML frontmatter（元資料）和 Markdown 指令。這是 Claude 判斷何時使用該 Skill 的關鍵。

Claude Code（Claude 程式碼工具）

Anthropic 提供的命令列工具，讓開發者可以直接從終端機將程式任務委派給 Claude。在這個情境中，用於分析和修復 Skill 檔案。

比喻說明 Analogy

Skill 就像餐廳的標準作業程序（SOP）

想像你開了一家連鎖餐廳。每個分店都有新員工，但你不能每次都親自教導。所以你會準備：

  • SKILL.md = 員工手冊主文（核心流程、服務標準）
  • scripts/ = 標準食譜（照著做就不會錯）
  • references/ = 食材供應商清單、過敏原資訊
  • assets/ = Logo、菜單模板、制服規格

新員工（新的 Claude 對話）拿到這份 SOP，就能立刻上手，不需要你從頭教起。

重點提示 Key Insight

為什麼你的第一個 Skill 會失敗？

三個最常見的原因：

  1. 檔案命名錯誤 — Claude 可能生成 Skill.md 而不是 SKILL.md（大小寫敏感）
  2. 參考檔案遺失 — SKILL.md 中提到的檔案實際上不存在
  3. 資料夾結構錯誤 — skill.md 沒有放在正確的根目錄

檔案結構公式 File Structure

skill-name/
├── SKILL.md ← 必須（主指令檔）
├── scripts/ ← 可執行程式碼
│ └── example.py
├── references/ ← 參考文件
│ └── guide.md
└── assets/ ← 輸出資源
└── template.png

YAML Frontmatter 格式：

---
name: my-skill-name
description: 清楚描述這個 Skill 做什麼，以及何時應該被觸發
---

實務應用 Practical Application

建構 Skill 的完整工作流程：

  1. 啟動 Skill Creator — 在 Claude 中激活技能創建模式
  2. 透過對話建構 — 描述你想要的功能
  3. 下載檔案 — 獲取生成的 ZIP
  4. 用 Claude Code 修復 — 分析並修正問題
  5. 正確打包 — 確保結構正確
  6. 上傳到帳戶 — 測試並迭代

Debug 技巧：

在 Cursor 或 VS Code 中使用 Claude Code 擴充套件，可以讓 AI 自動分析 Skill 檔案結構，找出缺失或格式錯誤的地方。

練習思考 Practice Questions

  1. 如果你要建構一個「會議紀錄摘要器」的 Skill，你會在 scripts/、references/、assets/ 各放什麼內容？
  2. 為什麼 SKILL.md 的 description 欄位如此重要？它與 Claude 何時觸發這個 Skill 有什麼關係？
  3. 當你的 Skill 在聊天中運作正常但上傳失敗時，你會先檢查哪三件事？

進階提示 Advanced Tips

Context Window（上下文視窗）是公共資源

Skill 與系統提示、對話歷史、其他 Skill 的元資料共享上下文視窗。所以：

  • 保持 SKILL.md 精簡（建議 500 行以內）
  • 只加入 Claude 不知道的資訊
  • 用 references/ 存放詳細文件，按需載入

本文重點摘要 Key Takeaways

  • Claude Skills 是擴展 AI 能力的模組化套件，包含專業知識、工作流程和工具
  • SKILL.md 是每個 Skill 的核心，必須有正確的 YAML frontmatter 和清晰的 description
  • 常見失敗原因：檔案命名錯誤、參考檔案遺失、資料夾結構不正確
  • 使用 Claude Code + Cursor/VS Code 可以有效 debug 和修復 Skill 檔案
  • 建構 Skill 是一個迭代過程——預期會遇到問題，準備好修復它們
  • 保持 Skill 精簡，善用 Progressive Disclosure（漸進式揭露）設計原則
