Part I — CoWork 是什麼？為什麼它不只是聊天機器人

What is CoWork & Why It Matters

CoWork 是執行者，不是建議者。標準的 Claude 會告訴你「這是整理發票的方法」，但 CoWork 會直接說「我已經按日期排好你的發票，把供應商名稱和金額拉進試算表，還標記了三張重複的。要我刪掉嗎？」

"Standard Claude gives you advice. CoWork executes the task."

CoWork 在 2026 年 1 月推出，最初僅限 Claude Max 訂閱者（$100-200/月）使用 macOS 版本。2026 年 2 月 10 日擴展到 Windows，並開放給所有付費方案（Pro $20/月起）。它運行在虛擬機器（VM）中，使用 Claude Opus 4.6 模型，擁有 100 萬 Token 的 Context Window。

"It runs on Claude Opus 4.6, with a 1-million-token context window — large enough to process entire projects in a single pass."

一個關鍵的背景：Anthropic 團隊用 Claude Code 在大約一週半的時間內建造了 CoWork。這本身就是 AI 遞迴改進（Recursive Improvement）的活生生案例 — AI 建造工具讓更多人能使用 AI。

"Claude Code wrote all of Claude Cowork. Can we all agree that we're in at least somewhat of a recursive improvement loop here?"

💡 Analogy

想像你有三種等級的助理：

Claude Chat = 電話顧問，你問問題，他給建議，但你得自己動手。

CoWork = 到場的助理，你描述結果，他直接在你辦公室裡完成工作。

Claude Code = 同一個助理但住在終端機裡，專門給開發者用。

Part II — Level 1: File Management（檔案管理）

Start Here: Your First Real Task

CoWork 在虛擬機器中運行。Claude 無法存取你沒有明確解鎖的資料夾以外的任何檔案。點擊資料夾圖示，選擇一個資料夾，那就是邊界。這是最基本但最能立即證明價值的起點。

"CoWork runs in a virtual machine. Claude cannot access files outside the folder you explicitly unlock."

三個立即證明概念的任務：

The Cleanup：「整理我的下載資料夾。按類型分組：圖片、PDF、安裝程式。建立命名資料夾。」手動 45 分鐘的 200 檔案資料夾，CoWork 四分鐘以內完成。

The Renamer：「根據圖片內容重新命名這個資料夾中的每個截圖。」Claude 在命名前會先讀取每張圖片。

The Data Analyst：「掃描這個發票資料夾。建立一個包含日期、供應商和金額的 CSV。」從雜亂的資料夾中產出結構化輸出。

"A 200-file folder that takes 45 minutes manually runs in under four."

🛠️ Quick Start（三步上手）
  1. 到 claude.com/download 下載 Desktop App
  2. 開啟 App，點擊頂部的 CoWork 標籤
  3. 點擊資料夾圖示，選擇工作目錄

建議先建立一個 Claude_Playground 測試資料夾，放入混合的假檔案來練習。

📚 Safety by Design

Deletion Protection（刪除保護）：CoWork 在移除任何檔案前都會要求確認。沒有靜默刪除。

VM Isolation（虛擬機隔離）：CoWork 在容器化環境中運行，與主作業系統隔離。你必須明確授權每個資料夾的存取。

Windows 版本額外限制：檔案存取僅限使用者個人資料夾，無法存取如 C:\git 等目錄。

Part III — Level 2: Web Agent（瀏覽器代理）

Claude in Chrome Extension Unlocks Web Access

安裝 Claude in Chrome 擴充功能後，CoWork 獲得網頁存取能力。它會開啟一個視覺上與你正常分頁不同的專用瀏覽器，像人類一樣操作 — 滾動、點擊、閱讀、回報。

"It opens a dedicated browser visually distinct from your normal tabs and operates it like a human."

三個值得立即執行的使用案例：

Deep Research：「去 X 和 Y 網站。拉出前三個 AI Agent 故事。整理成備忘錄。」兩小時的分頁切換，一個 Prompt 搞定。

Competitor Intel：「去 [URL]。截圖他們的定價頁面。把功能集整理成比較表。」

Sentiment Analysis：「找到這個 YouTube 頻道的最新影片。讀留言。給我整體情緒分析。」

"Two hours of tab-switching, done in one prompt."

對於你每天使用的網站（Gmail、CRM），選擇「always allow」可以移除每次的權限提示。

"For sites you use daily — Gmail, your CRM — choose 'allow always.'"

⚠️ Prompt Injection Risk

瀏覽器自動化有明確的安全警告：網站中隱藏的程式碼可能會「竊取你的數據、注入惡意軟體，或接管你的系統」。

Mitigation：CoWork 使用 VM 隔離，並且你控制哪些 MCP 連接和權限頻率。在擴展存取之前，仔細評估你對某個 MCP 或網站的信任程度。

💡 原文缺失的關鍵補充

原文作者將 CoWork 定位為 Pro ($20/mo) 就能使用，但實際上 CoWork 的 Token 消耗遠高於普通 Chat。Pro 用戶會很快遇到用量上限。

Anthropic 官方建議重度使用者考慮 Max 方案：5x ($100/mo) 約 225+ 訊息/5小時，20x ($200/mo) 約 900+ 訊息/5小時。

Part IV — Level 3: Connectors（連接器）

MCP: The Bridge Between Claude and Your Apps

CoWork 透過 MCP（Model Context Protocol，模型上下文協議）與外部應用程式對話。把它想成 Claude 和你數據之間的即時橋樑。

"Think of it as a live bridge between Claude and your data."

一鍵連接（Settings > Connectors）：Google Drive、Gmail、Google Calendar。一個 Prompt 就能串聯三個 App：「檢查我的行事曆找到下一個空檔。拉出我跟 John 的最後一封 email 看看我們需要討論什麼。起草一個會議邀請。」

"One prompt. Three apps. Zero tab-switching."

對於未列出的工具（Notion、Slack、Salesforce），搜尋「[App Name] MCP server」。進階用戶可以在 Desktop Settings 中編輯 config.json。非技術用戶應該找 Community Skills 來自動打包這些連接。

"For tools not listed — search '[App Name] MCP server.'"

📚 MCP 架構速覽

MCP 是 Anthropic 推出的開放標準，類似 USB-C 概念：建一次連接器，可跨 Claude、ChatGPT、Gemini 等平台使用。

架構：Host App (Claude Desktop) → MCP Client → MCP Servers（提供 Tools / Resources / Prompts）

注意：每個 MCP 工具的 Schema 定義都會佔用 Context Window。多個 MCP Server = 模型思考空間減少。這是選擇性啟用而非全部啟用的原因。

🛠️ 連接自訂 MCP

Settings > Add a Custom Connector → 提供 MCP 名稱和 Server URL → (Optional) OAuth credentials

推薦的聚合 MCP：Rube（rube.app）— 透過單一 MCP server 連接 500+ apps，減少 Context Window 消耗。

Part V — Level 4: Skills & Plugins（技能與插件）

The Real Multiplier: Teach Once, Execute Forever

Skill 是已儲存、可重複的工作流。你不再每次從頭 Prompting，而是教 Claude 一次流程，它按指令執行。原文作者給了一個例子：每週一把影片逐字稿轉為 X 貼文和 Newsletter 簡介，90 秒完成。

"A Skill is a saved, repeatable workflow. You teach Claude a process once, and it executes on command."

使用現有 Skills：Settings > Capabilities > Add Skill，上傳 .zip 檔案（如 smithy.ai 提供的）。像「SEO Blog Writer」或「Resume Tailor」這樣的 Skills 立即可用。

建立自己的 Skill：手動執行一次任務 → 調整到結果完美 → 問 Claude：「Create a Skill from this session so I can repeat it later.」

"You stop prompting from scratch every time."

2026 年 1 月 30 日，Anthropic 更進一步推出了 Plugin 系統。Plugin 是 Skills + MCP Connectors + Slash Commands + Sub-agents 的打包組合。11 個官方 Plugin 涵蓋 Sales、Legal、Finance、Marketing、Data Analysis 等領域。

"Plugins let you go further: tell Claude how you like work done."

📚 Skills vs Plugins vs MCP

這三層是一個堆疊（Stack），不是互相競爭的選項：

MCP = 傳輸層（Transport）。處理 Claude 和外部系統之間的連接。

Skills = 指令層（Instructions）。告訴 Claude 如何做事。

Plugins = 產品層（Product）。把兩者加上 Commands 和 Sub-agents 打包成可安裝的「App」。

💻 Plugin 檔案結構
plugin-name/
├── .claude-plugin/
│   └── plugin.json    # Metadata
├── .mcp.json          # MCP Connectors
├── commands/           # Slash commands
│   ├── call-prep.md
│   └── review.md
└── skills/            # Domain knowledge
    ├── analysis.md
    └── functions.py
🛠️ 安裝 Plugins

CoWork 內建：Settings > Plugins > Browse & Install

上傳自訂：Settings > Plugins > Upload Plugin

使用指令：輸入 / 或點擊 + 查看可用的 Slash Commands

Part VI — Level 5: Expert Tactics & Known Limitations

Advanced Strategies + What CoWork Can't Do Yet

Parallel Processing（平行處理）：開啟多個 CoWork 分頁。Tab 1 做研究、Tab 2 整理檔案、Tab 3 起草 email。你管理，它執行，全部同時進行。

"You manage. It executes. All simultaneously."

Plan Mode（計畫模式）：對複雜任務，先讓 Claude 寫計畫。審查後再說「Proceed」。這能保護你的用量配額並在錯誤複合之前攔截。

"Ask Claude to write a plan first. Review it. Then say 'Proceed.'"

Self-Verification（自我驗證）：在任何 Prompt 後加上：「After you finish, verify the output and flag anything missed.」強制第二輪檢查。

Project Integration：把你的 Claude Projects（品牌聲音、風格指南、業務背景）直接拉進 CoWork。點擊 + > Include Project。CoWork 就會用你的特定知識庫來執行任務。

⚠️ Current Limitations

No Cross-session Memory：新視窗不記得上一個。需要連續性就存成檔案或 Skill。

App Must Stay Open：關閉 Desktop App = 任務停止。

Quota Burns Fast：CoWork 的多步驟思考消耗遠超普通 Chat。重度使用者應考慮 Max 方案。

Still Makes Mistakes：瀏覽器中的誤點擊、檔案的誤讀都會發生。一定要 review output。

No Sharing：Sessions 無法與他人共享。

Research Preview：這仍是研究預覽版。不要用於受監管的工作負載（regulated workloads）。

Key Takeaways

Practice Questions

Q1 Concept CoWork 與標準 Claude Chat 的核心架構差異是什麼？為什麼這個差異對「執行」而非「建議」如此重要？

核心差異在於Agentic Architecture（代理架構）。Chat 是「一問一答」模式，每次回應是獨立的。CoWork 能夠：(1) 自主規劃多步驟任務，(2) 直接讀寫本地檔案系統，(3) 協調多個 Sub-agents 平行工作，(4) 透過 MCP 連接外部服務。

這些能力合在一起意味著 CoWork 不需要你在中間「手動傳遞」結果 — 它可以自己完成從「讀取發票」到「建立試算表」到「標記異常」的整個鏈條。這就是從 Advisor 到 Executor 的質變。

Q2 Industry Application 在 GMP 製藥環境中，CoWork 的哪些功能可能有價值？哪些功能因為「Research Preview」狀態而絕對不能用於生產？

有潛在價值的：檔案整理（SOP 文件分類、Batch Record 歸檔）、研究彙整（法規更新監控、競爭對手 CDMO 情報）、會議準備（從 email 和行事曆自動生成 pre-read）。這些都是非受監管的輔助性工作。

絕對不能用的：任何需要 21 CFR Part 11 合規的工作 — Batch Record 數據輸入、偏差調查報告撰寫、品質決策支援。Anthropic 自己明確說「Do not use Cowork for regulated workloads」，而且目前沒有 Audit Log、Compliance API、或 Data Export 功能。

策略是：先用在個人生產力（Copilot Mode），等 Anthropic 加上企業合規功能後再評估 GMP 場景。

Q3 Critical Thinking 原文作者說「paying the same $20, the difference is entirely in what they know」。這個說法有什麼問題？你會如何更準確地框架這個投資決策？

原文作者簡化了成本結構。$20 Pro 方案的 CoWork 用量會「很快耗盡」（Anthropic 和多個評測都證實）。真正的 power user 需要 $100-200/月的 Max 方案。所以實際差異不只是「知識」，還有10 倍的價格差距。

更準確的框架：這是一個ROI 計算題。如果你每週花 5 小時在 CoWork 能自動化的任務上，而你的時薪是 $50，那每月節省 $1000 的生產力。即使 $200/月的 Max 方案也只佔節省值的 20%。但如果你只是偶爾整理下載資料夾，$20 Pro 就夠了。關鍵是先用 Pro 測試你的實際使用頻率，再決定是否升級。
